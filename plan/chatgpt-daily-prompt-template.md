# ChatGPT Daily Prompt Template

用途：每天在 ChatGPT 网页版新开对话时，复制本模板，把 Day 编号、主题、关键词和验收题替换成当天内容。

注意：新开的 ChatGPT 对话不知道本仓库里的计划，也不知道之前的对话约定。必须把下面的“严格执行规则”和当天模板一起复制过去。

## 严格执行规则

每天发给 ChatGPT 时，开头必须包含这一段：

```text
请严格按我下面给出的 Day 计划执行。

你不要自由扩展成其他课程，也不要提前进入后面阶段。
你不要一次性讲完整天内容。
你必须按阶段进行：

1. 先只确认今天主题、学习目标、总时长、阶段安排。
2. 然后停下来，问我是否开始第 1 阶段。
3. 我回复“继续”后，你才讲第 1 阶段。
4. 每个阶段结束后都停下来，等我回复“继续”。
5. 到验收阶段时，你只根据当天验收题检查我是否通过。

如果我问了超出今天主题的问题，你可以简短回答，但要提醒我：这个问题属于后面阶段，今天先不展开。
```

今天只围绕一个核心问题：

电脑为什么能运行程序？

请按下面时间安排带我学习：

第 1 阶段：概念理解，60 分钟

请用非常通俗的语言解释这 5 个词：

- 操作系统
- CPU
- 内存
- 硬盘
- 程序

每个词都按这个格式解释：

- 它是什么
- 它解决什么问题
- 我在哪里会遇到
- 一个生活类比
- 一个电脑里的真实例子

请特别帮我区分：

- 硬盘 = 长期保存
- 内存 = 临时运行
- CPU = 真正执行计算
- 操作系统 = 管理者
- 程序 = 要被执行的任务或指令集合

第 2 阶段：现实类比，30 分钟

请用一个“书房/办公桌”的类比解释：

- 硬盘像什么
- 内存像什么
- CPU 像什么
- 操作系统像什么
- 程序像什么

然后解释：

打开一个软件时，大概发生了什么？

请用这个流程解释：

软件文件在硬盘里
    ↓
操作系统把它加载到内存
    ↓
CPU 执行它的指令
    ↓
程序开始运行

第 3 阶段：任务管理器观察练习，60 分钟

请指导我打开任务管理器：

Ctrl + Shift + Esc

然后让我观察这些内容：

- 进程
- CPU
- 内存
- 磁盘

请让我完成这些观察任务：

1. 当前 CPU 使用率大概是多少？
2. 当前内存使用率大概是多少？
3. 哪个程序占用内存比较多？
4. 打开 VS Code 后，任务管理器里有没有出现相关进程？
5. 关闭 VS Code 后，这些进程有什么变化？
6. 打开浏览器后，CPU 和内存有没有变化？
7. 关闭浏览器后，内存占用有没有下降？

请解释：

- 为什么打开程序后内存会上升？
- 为什么关闭程序后内存可能下降？
- 为什么 CPU 使用率会忽高忽低？
- 磁盘使用率表示什么？
