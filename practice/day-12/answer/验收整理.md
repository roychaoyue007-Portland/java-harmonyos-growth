# Day 12 验收与整理

完成日期：2026-06-28  
验收主题：前台程序、后台程序、服务、监听与本机端口  

## 一、验收结论

**最终结论：Day 12 通过。**

整体已经能够理解：

- 前台程序和后台程序的基本区别。
- 服务是更适合长期后台运行、由系统管理的程序。
- `localhost` 表示当前这台电脑。
- 停止服务、停止进程、关闭窗口会产生不同影响。
- 监听端口和服务入口之间有基本关系。

主要需要修正的点是：

```text
前台程序不一定等于“有窗口”；服务不是永远运行；监听端口表示某个进程已经在等待连接，不是连接来了才启动进程。
```

## 二、验收作答与评分

### 题目 1：用自己的话说明前台程序和后台程序有什么区别。

原回答：

> 前台是有窗口展示的进程，后台没有窗口展示，只在后台运行的进程

评分：8 / 10。

整理：前台程序是当前需要用户直接交互、占用窗口或终端的程序；后台程序是在用户不直接操作时仍然继续运行的程序。前台程序不一定必须有图形窗口，命令行里占用终端的程序也可以算前台程序。

### 题目 2：服务是什么？它和普通进程有什么关系？

原回答：

> 服务是一种被系统直接管理的特殊进程；它一直是后台进程，没有普通进程的前后变化，然后是被系统托管的

评分：8 / 10。

整理：服务是由系统或服务管理器管理、适合长期后台运行的程序。服务通常也需要由进程承载或执行，但它不是永远运行，也可以处于正在运行、已停止、正在启动或正在停止等状态。

### 题目 3：什么叫“监听端口”？为什么说监听表示正在等待连接？

原回答：

> 服务开启后，等待运行命令的端口；因为监听的时候，端口已经创建了，就是需要连接后，才开始运行对应的进程

评分：7 / 10。

整理：监听端口表示某个进程已经打开了一个端口，并正在等待连接请求。通常不是“连接后才开始运行进程”，而是进程已经在运行，连接来了以后由这个进程处理请求。

### 题目 4：`localhost` 表示什么？

原回答：

> 本地，本计算机

评分：10 / 10。

整理：`localhost` 表示当前这台电脑，也叫本地主机。

### 题目 5：`localhost:端口` 大致表示什么？

原回答：

> 本地端口

评分：8 / 10。

整理：`localhost:端口` 表示当前这台电脑上的某个服务入口。端口像同一台电脑上的服务窗口编号。

### 题目 6：`Get-Process` 可以帮我观察什么？

原回答：

> 当前计算机的进程

评分：10 / 10。

整理：`Get-Process` 可以观察当前电脑正在运行的进程。

### 题目 7：`Get-Service | Select-Object -First 10` 可以帮我观察什么？

原回答：

> 应该是当前按某个顺序排序的前十的服务，这个排序原则不知道

评分：8 / 10。

整理：这个命令显示服务列表中的前 10 条，用来快速观察服务名称、显示名称和状态。今天重点不是排序规则，而是认识服务列表和服务状态。

### 题目 8：`Get-NetTCPConnection -State Listen | Select-Object -First 10` 可以帮我观察什么？

原回答：

> 当前有哪些监听服务

评分：8 / 10。

整理：这个命令可以观察当前处于 Listen 状态的 TCP 连接记录，也就是当前本机有哪些端口正在被进程监听。更准确地说，它显示监听连接/监听端口，不一定直接等同于“服务”。

### 题目 9：关闭窗口、停止进程、停止服务分别可能意味着什么？

原回答：

> 关闭窗口-前台进程退出；停止进程-后台进程结束；停止服务-服务停止，监听取消，端口回收

评分：9 / 10。

整理：关闭窗口可能导致前台程序退出；停止进程会结束某个正在运行的程序实例；停止服务会通过服务管理机制停止服务，服务对应的监听端口通常也会消失。不要随意停止系统关键进程或服务。

### 题目 10：为什么有些程序关掉窗口就结束，而有些服务能一直在后台等待请求？

原回答：

> 因为这些程序设计的是只有前台进程，没有对应的后台进程。服务本身就属于后台进程，是特殊的后台进程，启动之后就是在等待请求再执行对应的服务

评分：8 / 10。

整理：有些程序设计成依赖窗口或终端交互，窗口关闭后进程就结束；服务则通常由系统管理，适合长期后台运行，可以在没有用户直接操作时继续等待请求。

## 三、最终知识整理

### 前台程序

```text
当前需要用户直接交互、占用窗口或终端的程序。
```

### 后台程序

```text
不直接显示在用户面前，但仍然继续运行的程序。
```

### 服务

```text
由系统或服务管理器管理、适合长期后台运行的程序。
```

### 监听

```text
某个进程打开端口并等待连接请求。
```

### localhost

```text
当前这台电脑。
```

## 四、薄弱点与后续复习

| 薄弱点 | 当前正确理解 | 后续复习方式 |
| --- | --- | --- |
| 前台程序 | 不一定有图形窗口，也可以是占用终端的程序 | 用命令行程序举例 |
| 服务 | 通常后台运行，但也可以停止 | 观察服务状态 |
| 监听端口 | 进程已经运行并等待连接 | 不说成“连接后才启动进程” |
| localhost:端口 | 当前电脑上的某个服务入口 | 记成“本机 + 服务窗口编号” |
