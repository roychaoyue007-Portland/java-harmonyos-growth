# Day 06：客户端、服务端、网络入口

3. 用来表明服务端在哪个地址，客户端可以通过 IP 地址找到服务端在网络上的地址。

## 修正版答案

### 1. 什么是客户端？请举 2 个例子。

客户端是与用户交互、主动发起请求的一方。常见例子有浏览器、鸿蒙 App、手机 App、桌面软件。

### 2. 什么是服务端？它主要负责什么？

服务端是接收客户端请求、处理数据并返回结果的一方。它通常在后台运行，不直接面对普通用户，主要负责处理业务逻辑、读写数据、返回响应结果。

### 3. IP 地址主要用来解决什么问题？

IP 地址用来标识网络中的一台设备。客户端可以通过服务端的 IP 地址找到服务端所在的机器。它主要解决“去哪台机器”的问题。

### 4. 端口主要用来解决什么问题？

端口用来区分一台机器上的不同服务。只知道 IP 只能找到机器，端口可以继续定位到这台机器上的具体服务。

### 5. HTTP 是什么？它在客户端和服务端之间起什么作用？

HTTP 是客户端和服务端之间常用的一种通信协议。它规定客户端如何发送请求、服务端如何返回响应，让双方按照共同规则交流。

### 6. 浏览器输入一个网址后，客户端和服务端大概怎么通信？

浏览器作为客户端，会先根据网址找到服务端地址。网址通常会被解析成 IP 地址，然后浏览器连接服务端对应端口，按照 HTTP 或 HTTPS 协议发送请求。服务端接收请求并处理，再返回响应，浏览器收到响应后展示结果。

### 7. 为什么同一台服务器可以运行多个服务？端口在这里有什么作用？

一台服务器本质上是一台机器，它可以同时运行多个程序或服务。不同服务需要用不同端口区分。IP 负责找到这台机器，端口负责找到这台机器上的具体服务。

### 8. 在一个未来的全栈项目中，鸿蒙 App、Java 后端、HTTP 分别大概对应什么角色？

鸿蒙 App 是客户端，Java 后端是服务端，HTTP 是客户端和服务端之间通信的协议。鸿蒙 App 通过 HTTP 请求 Java 后端，Java 后端处理后通过 HTTP 返回结果。
