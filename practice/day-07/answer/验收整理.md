## 三、已经掌握的内容

- 文件保存在硬盘的某个路径中，目录可以包含文件和其他目录。
- 硬盘负责长期保存，内存提供程序运行时的临时工作空间。
- 程序是静态文件，启动后操作系统创建进程。
- 线程属于进程，并负责执行具体任务。
- IP 用来定位机器，端口用来定位机器上的具体服务。
- HTTP 是客户端和服务端共同遵守的通信规则。
- 能使用 `javac`、`java`、`ipconfig` 和 `curl.exe` 完成基础观察。

## 四、需要修正的表达

### 1. PATH

原理解中的 `pass` 应为 `PATH`。

**正确理解：**环境变量是操作系统保存的一组“名称—值”配置。`PATH` 是其中一个特殊的环境变量，它保存了一串目录。输入 `java` 时，系统会按顺序到这些目录中寻找 `java.exe`。

```text
输入 java
    → PowerShell 先检查命令
    → Windows 按 PATH 中的目录依次查找 java.exe
    → 找到后启动它
```

### 2. 编译、解释执行和运行时

- **编译**：把一种代码转换成另一种更适合执行的形式。`javac` 把 `.java` 源代码编译成 `.class` 字节码。
- **解释执行**：运行过程中读取代码或中间表示，并执行对应指令。JVM 可以解释执行 Java 字节码。
- **运行时**：程序运行时依赖的环境和支持系统。Java 的主要运行时是 JVM 及相关类库。

编译不是“解释”。编译更像提前翻译并产生新文件；解释执行更像边读取边执行。

### 3. 程序、进程和线程

进程不是“存在于任务管理器中”。进程存在于操作系统管理的运行环境中，任务管理器只是把它显示出来。

```text
程序：硬盘上的静态文件
进程：程序启动后产生的运行实例
线程：进程中执行任务的单位
任务管理器：观察和管理进程的工具
```

### 4. 客户端与服务端

客户端通常负责与用户交互、发出请求和展示结果，不一定负责长期存放业务数据。服务端通常负责接收请求、处理业务、读写数据并返回响应。

### 5. 版本号

版本号不是表达“程序能够干什么”，而是标识软件的具体发布版本，用来判断功能差异、兼容性、更新与问题修复情况，例如 `Java 17` 和 `Java 21`。

## 五、第一周知识地图整理

```text
源代码文件保存在硬盘的某个路径中
    → 路径让系统准确定位文件
用户通过命令或脚本操作文件
    → 命令告诉计算机要执行什么动作
操作系统根据 PATH 找到 javac、java 等程序
    → PATH 告诉系统应去哪些目录寻找命令
javac 把 Java 源代码编译成字节码
    → JVM 需要能够识别的 .class 字节码
java 命令启动 JVM，并加载字节码
    → JVM 提供 Java 程序运行所需的环境
操作系统为 JVM 创建进程并分配内存和 CPU 时间
    → 运行中的程序需要资源
进程中的线程执行具体任务
    → CPU 实际执行线程中的指令
客户端根据域名获得服务端 IP
    → IP 用来找到网络中的目标机器
客户端连接目标端口
    → 端口用来找到机器上的具体服务
客户端按照 HTTP/HTTPS 规则发送请求
    → 服务端处理请求并返回响应
```

## 六、综合题修正版

一个 Java 客户端程序最初以 `.java` 源代码文件保存在硬盘的某个路径中。执行 `javac` 后，编译器把源代码转换为 `.class` 字节码。执行 `java` 命令时，系统通过 PATH 找到 Java，启动 JVM；操作系统为 JVM 创建进程、分配内存和 CPU 时间，进程中的线程开始执行字节码对应的任务。当程序需要访问网络服务时，客户端先根据域名获得服务端 IP，通过端口找到目标机器上的具体服务，再按照 HTTP 或 HTTPS 规则发送请求。服务端处理请求后返回响应，客户端接收并使用结果。
