## 二、首次验收作答与评分

### 1. 终端和 Shell 的区别

原回答：

> 终端是用户的输入入口，Shell 是计算机的内部可执行的用户输入的依据。

得分：6 / 10。

整理：终端负责接收输入和显示结果；Shell 是读取、解析并执行命令的程序。Shell 不是“执行依据”，而是实际处理命令的软件。

### 2. PowerShell 属于什么

原回答：

> 是 Shell 的具体表现，因为它接手并解析用户的输入，执行对应的命令。

得分：9 / 10。

整理：PowerShell 是一种具体的 Shell，同时也是脚本环境，不是终端，也不是操作系统。

### 3. 拆解命令和参数

原回答：

> `Get-ChildItem` 是具体的命令，`-Path` 是参数名，`.` 是参数值，代表当前路径，`-File` 是参数值，代表文件类型。

得分：7 / 10。

整理：

```text
Get-ChildItem：命令名
-Path：参数名
.：-Path 的参数值，表示当前目录
-File：开关参数，只筛选文件，本身不需要参数值
```

### 4. `Get-Command java` 的作用

原回答：

> 查到 java 这个命令的一些信息，如类型是外部程序。

得分：7 / 10。

整理：它可以调查 `java` 的命令类型、来源、版本和实际路径，帮助确认 PowerShell 最终会执行哪个 `java.exe`。

### 5. `Get-Help Get-ChildItem -Examples` 的作用

得分：3 / 10。

### 6. Cmdlet 与外部程序的区别

原回答：

> PowerShell 命令主要是计算机本地的，`java.exe` 是扩展的。

得分：5 / 10。

整理：`Get-ChildItem` 是由 PowerShell 环境提供并执行的 Cmdlet；`java.exe` 是独立安装的外部可执行程序。PowerShell 通常需要通过命令解析和 PATH 查找才能定位外部程序。

### 7. PATH 的作用

原回答：

> 找到对应外部命令的应用程序。

得分：8 / 10。

整理：PATH 是一个环境变量，它保存一组目录。执行外部命令时，Shell 可以到这些目录中查找对应的可执行程序。

### 8. `java -version` 的执行流程

原回答：

> 终端接收信息，PowerShell 解析，判断是外部程序，通过 PATH 去找，找到 `java.exe` 后，识别 `-version` 指令，输出版本号。

得分：10 / 10。

整理后的流程：

```text
终端接收输入
    → PowerShell 解析命令名和参数
    → 判断 java 是外部程序
    → 通过 PATH 查找 java.exe
    → 启动 java.exe，并把 -version 作为参数传给它
    → Java 程序输出版本信息
    → 终端显示结果
```

## 四、最终知识整理

### 终端、Shell 与 PowerShell

```text
终端：输入和显示的界面
Shell：读取、解析并执行命令的程序
PowerShell：Windows 上常用的一种具体 Shell
```

### 命令与参数

```text
命令：说明要做什么
参数：说明对谁做、怎样做
参数值：参数实际接收的内容
开关参数：通过是否出现来开启一种行为，通常不需要额外参数值
```

### Cmdlet 与外部程序

```text
PowerShell Cmdlet
    → 由 PowerShell 环境提供并执行

外部程序，例如 java.exe
    → 是硬盘上的独立可执行文件
    → Shell 需要先定位它
    → PATH 可以提供查找目录
```

### 两个调查工具

- `Get-Command`：调查命令是什么类型、来自哪里、最终会执行哪个程序。
- `Get-Help`：学习 PowerShell 命令的语法、参数和示例。
