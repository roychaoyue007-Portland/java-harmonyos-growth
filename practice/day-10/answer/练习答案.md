## 一、练习前检查

练习前先确认当前目录：

```powershell
Get-Location
```

如果当前目录下没有 `work` 文件夹，先创建：

```powershell
New-Item -ItemType Directory -Path .\work
```

注意：练习应该在学习目录中完成，不要在系统目录中随意写文件。

## 二、命令练习记录

### 1. `Write-Output "hello"`

命令：

```powershell
Write-Output "hello"
```

观察：

```text
终端显示 hello。
```

整理：这是一条正常输出。命令成功执行后，把 `hello` 作为结果输出到终端。

### 2. `Get-ChildItem | Select-Object Name`

命令：

```powershell
Get-ChildItem | Select-Object Name
```

观察：

```text
终端显示当前目录下项目的 Name 列。
```

整理：

```text
Get-ChildItem       获取当前目录下的项目
|                   把结果交给右边
Select-Object Name  只保留 Name 属性
```

这条命令重点练习管道：左边命令的输出成为右边命令的输入。

### 3. `Get-ChildItem | Where-Object Length -GT 0`

命令：

```powershell
Get-ChildItem | Where-Object Length -GT 0
```

观察：

```text
终端显示 Length 大于 0 的项目。
```

整理：初学阶段可以把它理解为筛选有内容的文件。严格来说，`Where-Object` 会根据对象的 `Length` 属性进行筛选。

### 4. `Get-ChildItem | Out-File .\work\file-list.txt`

命令：

```powershell
Get-ChildItem | Out-File .\work\file-list.txt
```

观察：

```text
结果被写入 .\work\file-list.txt。
终端上不一定直接显示文件列表。
```

整理：这条命令体现重定向或输出位置改变：结果不只是显示在终端，而是保存到文件中。

### 5. `Get-Content .\work\file-list.txt`

命令：

```powershell
Get-Content .\work\file-list.txt
```

观察：

```text
终端显示 file-list.txt 中保存的内容。
```

整理：`Get-Content` 用来读取文件内容。它把文件里的文本读出来，再显示到终端。

### 6. 故意输入不存在的命令

命令：

```powershell
not-a-real-command
```

观察：

```text
PowerShell 显示错误信息，提示命令无法识别。
```

整理：这不是正常输出，而是错误输出。错误输出的作用是告诉我命令哪里失败了。

## 三、重点观察整理

### `|` 左边的输出怎样成为右边的输入？

管道 `|` 会把左边命令产生的结果交给右边命令继续处理。

```powershell
Get-ChildItem | Select-Object Name
```

理解：

```text
先拿到当前目录项目，再只选择 Name。
```

### 终端显示和写入文件有什么不同？

```text
终端显示：结果只是出现在屏幕上，方便马上查看。
写入文件：结果保存到文件里，以后还可以查看或提交。
```

### 正常输出和错误输出有什么不同？

```text
正常输出：命令成功后的结果。
错误输出：命令失败或异常时给出的说明。
```

颜色可以作为辅助观察，但不能只靠颜色判断。真正要看它表达的是正常结果还是错误说明。
