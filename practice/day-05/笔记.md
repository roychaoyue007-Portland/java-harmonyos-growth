## 阶段 2：核心关系，30 分钟

今天要先记住这条关系：

```text
源代码：程序员写出来、保存在文件里的代码
编译：把源代码转换成更适合执行的形式
运行：让程序真正开始执行
解释执行：一边读取代码或中间表示，一边执行
运行时：程序运行时依赖的环境和支持系统
```

程序员写的是源代码。

电脑不能直接理解所有源代码。有些语言需要先编译，再运行；有些语言可以解释执行；有些语言会混合使用编译、解释和即时编译。

Java 的典型过程是：

```text
Hello.java 源代码
    -> javac Hello.java
    -> Hello.class 字节码
    -> java Hello
    -> JVM 加载并执行
    -> 程序真正运行
```

## 阶段 3：动手观察，60 分钟

在 PowerShell 中观察 Java 源代码从文件到运行的过程：

```powershell
javac Hello.java
java Hello
```

观察重点：

- `Hello.java`：人写的源代码文件。
- `javac Hello.java`：调用 Java 编译器进行编译。
- `Hello.class`：编译后生成的 Java 字节码文件。
- `java Hello`：启动 JVM，加载 `Hello.class`，让程序运行。

如果出现报错，可以从这些方向排查：

1. 源代码语法写错了。
2. 当前目录不对，PowerShell 找不到 `Hello.java`。
3. `javac` 命令不可用，说明 JDK 或 PATH 可能有问题。
4. 运行命令写错，例如不应该写成 `java Hello.class`。
5. 编译成功但运行失败，可能是类名、文件名或运行环境问题。

## 我的原始验收答案

1. 人为写出来的，机器无法直接阅读的代码
2. 因为源代码机器它不能运行,它本身只是一段静止的文件。
3. 编译主要解决的是将人为书写的代码转变为机器可以阅读的机器代码。
4. 将Hello.java文件编译成hello.class
5. 执行hello.class文件
6. .java文件是源代码,.class文件是经过编译后的机器代码。
7. 解释执行是指对边阅读边执行代码,和先编译再运行的区别是先把文件先编译好,比如说编译成一个虚拟机可执行的Java类文件,然后后续再统一运行。
8. 没有完全理解

## 我的验收答案整理

1. 源代码是程序员用编程语言写出来、保存在文件里的代码。它主要给人阅读和维护，电脑通常不能直接把它当成正在运行的程序。
2. 源代码保存在硬盘上时只是静态文件。程序真正运行，需要经过编译或解释执行，并由操作系统或运行时环境加载执行。
3. 编译主要解决“把人写的代码转换成更适合执行的形式”的问题。Java 中更准确地说，是把 `.java` 源代码编译成 JVM 能理解的 `.class` 字节码。
4. `javac Hello.java` 会调用 Java 编译器，把 `Hello.java` 编译成 `Hello.class`。
5. `java Hello` 会启动 JVM，加载 `Hello.class`，然后让程序真正运行起来。
6. `.java` 文件是人写的源代码；`.class` 文件是编译后得到的 Java 字节码。`.class` 不是普通机器代码，它需要 JVM 运行时环境来执行。
7. 解释执行是运行时一边读取代码或中间表示，一边执行。先编译再运行则是先把源代码转换成另一种形式，再启动运行过程。
8. 运行时是程序真正运行时依赖的一整套环境和支持。比如 Java 程序运行时需要 JVM 来加载 class 文件、执行字节码、管理内存、处理错误并提供基础能力。

### 补充：为什么 `.class` 不是普通机器代码？

Java 的 `.class` 文件里主要是字节码，不是 Windows 或 CPU 可以直接裸跑的普通机器码。

它需要 JVM 来加载和执行。JVM 就像 Java 程序和操作系统之间的一层运行环境。

### 补充：什么是运行时？

一句话：

```text
运行时 = 程序跑起来时，背后帮它执行和管理的环境。
```

Java 里的运行时重点可以先记成 JVM。

JVM 会负责：

- 加载 `.class` 文件。
- 执行 Java 字节码。
- 管理内存。
- 处理运行时错误。
- 提供 Java 程序运行需要的基础能力。
