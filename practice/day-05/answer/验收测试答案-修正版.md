## 修正版答案
### 1. 什么是源代码？

源代码是程序员用编程语言写出来、保存在文件里的代码。它主要给人阅读和维护，电脑通常不能直接把它当成正在运行的程序。

### 2. 为什么源代码只是文件，不等于程序已经运行？

因为源代码保存在硬盘上时只是静态文件。程序真正运行，需要经过编译或解释执行，并由操作系统或运行时环境加载执行。

### 3. 编译主要解决什么问题？

编译主要解决“把人写的代码转换成更适合执行的形式”的问题。Java 中，编译不是直接变成普通机器代码，而是把 .java 文件编译成 JVM 能理解的 .class 字节码。

### 4. Java 中 javac Hello.java 大概做了什么？

`javac Hello.java` 会调用 Java 编译器，把 `Hello.java` 源代码编译成 `Hello.class` 字节码文件。

### 5. Java 中 java Hello 大概做了什么？

`java Hello` 会启动 JVM，加载 `Hello.class`，然后让程序真正运行起来。

### 6. .java 文件和 .class 文件有什么区别？

`.java` 文件是人写的源代码；`.class` 文件是编译后得到的 Java 字节码。`.class` 不是普通机器代码，它需要 JVM 运行时环境来执行。

### 7. 什么是解释执行？它和先编译再运行有什么不同？

解释执行是运行时一边读取代码或中间表示，一边执行。先编译再运行则是先把源代码转换成另一种形式，再启动运行过程。实际语言可能混合使用编译、解释和即时编译。

### 8. 什么是运行时？为什么程序运行时需要运行时环境？

运行时是程序真正运行时依赖的一整套环境和支持。比如 Java 程序运行时需要 JVM 来加载 class 文件、执行字节码、管理内存、处理错误并提供基础能力。
