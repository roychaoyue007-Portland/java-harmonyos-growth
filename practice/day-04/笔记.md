# Day 04：安装、配置、环境变量

## 我的修正版理解

### 1. 安装是什么？

安装就是把软件需要的文件放到电脑硬盘上，让这个软件真正存在于电脑里。

安装以后，电脑里通常会有软件目录、可执行文件、开始菜单入口、卸载信息等。

一句话：安装解决的是“这个软件有没有放到电脑上”的问题。

### 2. 配置是什么？

配置就是告诉操作系统或软件应该怎么工作、去哪里找文件、按什么规则使用。

安装只是让软件存在，配置是让软件能更准确地被系统或用户使用。

比如 JDK 安装好了，只说明 JDK 文件在电脑里；如果要在 PowerShell 里直接输入 `java` 或 `javac`，还可能需要配置 PATH。

### 3. 安装和配置有什么区别？

安装是让软件作为文件存在于硬盘上。

配置是告诉系统或软件如何正确使用这些文件。

所以安装偏向“放好软件”，配置偏向“设好规则”。

### 4. 环境变量是什么？

环境变量是操作系统保存的一些全局配置信息。

它像一张系统级的“公共便签”，很多程序和命令行都可以读取它。

它属于操作系统层面的信息，是因为它不是某一个软件自己私有的设置，而是 Windows 统一维护、多个程序都可能使用的公共配置。

### 5. PATH 是什么？

PATH 是一个重要的环境变量。

它告诉 Windows：当我在 PowerShell 输入一个命令时，系统应该去哪些文件夹里找对应的程序。

比如输入：

```powershell
git --version
```

Windows 会根据 PATH 里的目录去找 `git.exe`。如果找到了，就能执行；如果找不到，就会提示命令无法识别。

### 6. 版本号是什么？

版本号是软件当前这一版的编号，不是软件名字。

比如：

```text
Git = 软件名字
2.45.1 = 版本号
```

不同版本可能有不同功能、不同命令、不同兼容性。所以开发时经常要确认版本号。

### 7. 为什么安装 JDK、Git 后还要配置环境变量？

因为安装 JDK、Git 只是让软件存在于硬盘上。

但是 PowerShell 输入 `java`、`javac`、`git` 时，系统需要知道这些命令对应的程序在哪里。

如果对应目录没有加入 PATH，软件虽然已经安装，但命令行仍然可能找不到它。

所以配置环境变量，尤其是 PATH，是为了让系统能找到这些命令。

### 8. `java -version` 和 `where.exe java` 有什么区别？

`java -version` 是观察当前能运行的 Java 版本。

`where.exe java` 是观察系统实际找到了哪个路径下的 `java.exe`。

如果电脑上有多个 Java，`where.exe java` 可以帮助我判断系统优先使用的是哪一个。

### 9. 如果 PowerShell 提示找不到命令，可能是什么原因？

可能原因有：

1. 软件没有安装。
2. 软件安装了，但命令目录没有加入 PATH。
3. PATH 路径写错了。
4. 修改 PATH 后，没有重新打开 PowerShell。
5. 命令名字输错了。

### 10. 今天我对“安装”和“配置”的区别有什么新理解？

安装是让软件存在，配置是让系统知道怎么使用它。

只安装软件，不代表命令行一定能找到它。

配置 PATH 后，系统才知道输入命令时应该去哪里找对应的程序。

## 技术词典

### 词：安装

一句话解释：安装是把软件需要的文件放到电脑硬盘上。

它解决什么问题：让软件真正存在于电脑里。

我在哪里会遇到：安装 JDK、Git、VS Code、浏览器、鸿蒙开发工具。

相关词：软件、硬盘、安装目录、卸载。

我是否亲手做过：做过。

### 词：配置

一句话解释：配置是告诉系统或软件应该怎么工作。

它解决什么问题：让软件能按照正确的规则被使用。

我在哪里会遇到：配置 PATH、配置 Git 用户名、配置 IDE、配置项目参数。

相关词：设置、环境变量、PATH、配置文件。

我是否亲手做过：做过。

### 词：环境变量

一句话解释：环境变量是操作系统保存的全局配置信息。

它解决什么问题：让系统和多个程序共享一些公共信息。

我在哪里会遇到：Windows 系统环境变量、PATH、JAVA_HOME、TEMP。

相关词：操作系统、配置、PATH、JAVA_HOME。

我是否亲手做过：初步观察过。

### 词：PATH

一句话解释：PATH 是告诉系统输入命令时去哪些文件夹里找程序的环境变量。

它解决什么问题：让 PowerShell 可以直接识别 `java`、`git` 等命令。

我在哪里会遇到：PowerShell、JDK 配置、Git 配置、Node.js 配置。

相关词：环境变量、命令、可执行文件、目录。

我是否亲手做过：观察过。

### 词：版本号

一句话解释：版本号是软件当前这一版的编号。

它解决什么问题：帮助确认软件功能、兼容性和排查问题范围。

我在哪里会遇到：`java -version`、`git --version`、软件关于页面、项目文档。

相关词：软件、更新、兼容性、命令。

我是否亲手做过：做过。

## 今日总结

今天最重要的是理解：

```text
安装 = 软件存在于硬盘上
配置 = 告诉系统或软件怎么使用
环境变量 = 操作系统保存的全局配置
PATH = 系统查找命令的路线
版本号 = 软件当前版本的编号
```

所以，安装 JDK、Git 后还要配置环境变量，是因为命令行不只需要“软件存在”，还需要“系统知道去哪里找它”。
