# Day 04 验收测试答案

主题：安装、配置、环境变量

说明：下面先保留我的原始回答，再补充一版修正版，方便以后复习。

## 1. “安装一个软件”是什么意思？

我的答案：

让软件安装在电脑硬盘上。

修正版：

安装一个软件，就是把软件需要的文件放到电脑硬盘上，让这个软件真实存在于电脑里。安装解决的是“软件有没有放到电脑上”的问题。

## 2. “配置一个软件”是什么意思？

我的答案：

告诉操作系统，这个软件能干什么，怎么用它，它在哪里。

修正版：

配置一个软件，就是告诉操作系统或软件应该怎么工作、去哪里找文件、按什么规则使用。它解决的是“软件应该怎么被正确使用”的问题。

## 3. 安装和配置有什么区别？

我的答案：

安装是让应用存在于硬盘上，静止地存在。配置告诉系统怎么运用它。

修正版：

安装是让软件作为文件存在于硬盘上；配置是告诉系统或软件如何正确使用这些文件。只安装不配置，软件可能存在，但命令行不一定能找到它。

## 4. 环境变量是什么？它为什么属于操作系统层面的信息？

我的答案：

不知道；因为它是全局的。

修正版：

环境变量是操作系统保存的一些全局配置信息。它属于操作系统层面的信息，是因为它不是某一个软件私有的设置，而是 Windows 统一维护、多个程序都可能读取的公共配置。

## 5. PATH 环境变量主要解决什么问题？

我的答案：

在全局环境找到对应的应用，为了使用它。

修正版：

PATH 环境变量主要解决“输入命令时，系统应该去哪些文件夹里找程序”的问题。它让 PowerShell 可以直接识别 java、git 这类命令。

## 6. 为什么安装了 Git，但 PowerShell 里输入 git 仍然可能失败？

我的答案：

可能没有配置环境。

修正版：

可能是 Git 没有安装成功，或者 Git 安装了但命令目录没有加入 PATH，也可能是 PATH 修改后 PowerShell 没有重新打开。

## 7. java -version 和 where java 分别是在观察什么？

我的答案：

前者观察版本，后者观察路径。有的时候需要使用 where.exe。

修正版：

java -version 用来观察当前能运行的 Java 版本；where.exe java 用来观察系统实际找到了哪个路径下的 java.exe。

## 8. 版本号有什么用？为什么开发时经常要确认版本号？

我的答案：

版本号相当于应用当前的名字，因为不同版本号会有不同的功能。

修正版：

版本号不是软件名字，而是软件当前这一版的编号。开发时确认版本号，是为了判断功能、兼容性和排查问题范围。

## 今日结论

Day 04 通过。今天最重要的是分清：安装是让软件存在，配置是让系统知道怎么使用它，PATH 是系统查找命令的路线。
