# Day 11 验收与整理

完成日期：2026-06-28  
验收主题：环境变量、PATH、作用范围、配置与命令查找  

## 一、验收结论

**最终结论：Day 11 通过。**

整体已经能够理解：

- 环境变量是键值对配置。
- PATH 的值是一串用分号分隔的目录。
- 系统级 PATH 面向所有用户，用户级 PATH 只面向当前用户。
- 命令行能不能找到 `java.exe`，取决于命令查找过程和 PATH 等配置。

主要需要修正的点是：

```text
环境变量不一定直接“调用到对应文件”；PATH 也不是普通配置项，而是 Shell 查找外部命令的位置清单。
```

## 二、验收作答与评分

### 题目 1：用自己的话说明环境变量是什么。

原回答：

> 是全局可使用的键值对，配置好了可直接调用到对应的文件

评分：8 / 10。

整理：环境变量是操作系统提供给程序或 Shell 读取的键值对配置。它不一定直接对应文件，也可能保存目录、开关、临时路径等信息。

需要记住：

```text
环境变量是系统提供给程序读取的键值对配置。
```

### 题目 2：PATH 是什么？它的值大概由什么组成？

原回答：

> PATH是一个键值对，大概是键-PATH，值-一串以;分割的很长的路径的字符串

评分：9 / 10。

整理：PATH 是一个环境变量。它的值是一串目录列表，Windows 中通常用英文分号 `;` 分隔。Shell 会根据这些目录查找外部命令。

需要记住：

```text
PATH 是 Shell 查找外部命令的位置清单。
```

### 题目 3：PATH 和 JAVA_HOME 有什么区别？

原回答：

> 作用不同

评分：6 / 10。

整理：

```text
JAVA_HOME：通常指向 JDK 安装目录。
PATH：告诉 Shell 去哪些目录找可执行文件，比如 java.exe。
```

需要记住：JAVA_HOME 说明 Java 装在哪里；PATH 决定命令行能不能直接找到 `java.exe`。

### 题目 4：用户级 PATH 和系统级 PATH 可能有什么不同？

原回答：

> 系统级是面向全部用户的，换一个用户也找得到。

评分：8 / 10。

整理：系统级 PATH 对整台电脑所有用户生效；用户级 PATH 只对当前用户生效。用户级和系统级的内容可能不同，合并后会影响当前 Shell 的命令查找。

### 题目 5：在 PowerShell 输入 `java` 后，电脑大致怎样找到 `java.exe`？

原回答：

> 跟据环境变量的配置按个去找

评分：8 / 10。

整理：PowerShell 收到 `java` 这个命令后，会先判断它是不是内置命令或当前可直接识别的命令。如果是外部程序，就会根据 PATH 中列出的目录逐个查找 `java.exe`，找到后执行。

### 题目 6：`Get-Command java` 和 `where.exe java` 分别可以帮助我观察什么？

原回答：

> java.exe的具体信息；java.exe的具体路径

评分：9 / 10。

整理：`Get-Command java` 可以查看 PowerShell 识别到的 `java` 命令信息；`where.exe java` 可以列出系统按 PATH 找到的 `java.exe` 具体路径。

### 题目 7：为什么“软件已经安装”不等于“命令行一定能找到它”？

原回答：

> 因为没有配置环境变量的话，命令行无法找到

评分：9 / 10。

整理：软件安装只说明文件存在于电脑中。命令行能不能通过命令名找到它，还取决于 PATH 是否包含可执行文件所在目录、命令名是否正确、终端是否读取到最新配置等。

### 题目 8：如果提示“命令不存在”或“无法识别命令”，你会按什么基础步骤检查？

原回答：

> 检查环境变量是否配置，是否配置正确

评分：7 / 10。

整理：基础检查步骤可以更完整：

```text
1. 检查命令有没有拼错。
2. 检查软件是否已经安装。
3. 找到可执行文件实际安装目录。
4. 检查 PATH 是否包含可执行文件所在目录。
5. 重新打开终端后再试。
6. 用 Get-Command 或 where.exe 验证。
```

## 四、薄弱点与后续复习

| 薄弱点 | 当前正确理解 | 后续复习方式 |
| --- | --- | --- |
| 环境变量 | 系统给程序读取的键值对配置 | 不把它简单理解成“调用文件” |
| PATH | 查找外部命令的位置清单 | 记住它的值是一串目录，不是某个软件 |
| JAVA_HOME vs PATH | JAVA_HOME 指向 JDK 目录，PATH 用来找可执行文件 | 用 `java.exe` 的查找过程复述一次 |
| 命令不存在排查 | 不只看环境变量，还要看拼写、安装、目录、终端刷新 | 按固定步骤检查 |
