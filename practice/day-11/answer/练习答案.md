# Day 11 练习答案：环境变量、PATH、作用范围与命令查找

完成日期：2026-06-28  
主题：环境变量、PATH、作用范围、配置与命令查找  

## 一、练习原则

今天只观察，不修改系统或用户环境变量。

不要执行修改 PATH、JAVA_HOME 或系统环境变量的操作。今天的目标是理解它们是什么，以及命令行怎样利用它们查找外部程序。

## 二、命令练习记录

### 1. `$env:PATH`

命令：

```powershell
$env:PATH
```

观察目标：查看当前 PowerShell 会话中的 PATH 值。

整理：PATH 的值通常是一串很长的目录列表，Windows 中多个目录之间常用英文分号 `;` 分隔。

需要记住：

```text
PATH 不是某一个程序，而是一串“去哪里找程序”的目录清单。
```

### 2. `$env:JAVA_HOME`

命令：

```powershell
$env:JAVA_HOME
```

观察目标：查看当前会话中的 JAVA_HOME 值。

整理：JAVA_HOME 通常指向 JDK 安装目录。它可能有值，也可能为空。为空时，不要立刻修改，先继续用 `Get-Command java` 和 `where.exe java` 观察命令是否能被找到。

### 3. `[Environment]::GetEnvironmentVariable('PATH', 'User')`

命令：

```powershell
[Environment]::GetEnvironmentVariable('PATH', 'User')
```

观察目标：查看用户级 PATH。

整理：用户级 PATH 只对当前用户有效。它可能包含用户自己安装的软件目录。

### 4. `[Environment]::GetEnvironmentVariable('PATH', 'Machine')`

命令：

```powershell
[Environment]::GetEnvironmentVariable('PATH', 'Machine')
```

观察目标：查看系统级 PATH。

整理：系统级 PATH 对这台电脑上的所有用户有效。它通常包含系统目录或面向全局安装的软件目录。

### 5. `Get-Command java`

命令：

```powershell
Get-Command java
```

观察目标：查看 PowerShell 实际识别到的 `java` 命令信息。

整理：如果能找到，通常会显示命令类型和路径等信息。它回答的是：PowerShell 现在能不能识别 `java`，以及识别到的是哪个命令。

### 6. `where.exe java`

命令：

```powershell
where.exe java
```

观察目标：查看 Windows 按 PATH 查找到的 `java.exe` 路径。

整理：它通常会列出一个或多个 `java.exe` 的具体路径。它回答的是：系统沿着 PATH 到底找到了哪些 `java.exe`。

## 三、重点观察整理

### PATH 本身就是环境变量

PATH 不是 Java 独有的配置，也不是一个软件名。它是操作系统环境变量中的一个键。

### PATH 的值由多个目录组成

PATH 的值大概长这样：

```text
目录1;目录2;目录3;...
```

Shell 查找外部命令时，会参考这些目录。

### 用户级和系统级 PATH 可能不同

```text
用户级 PATH：只对当前用户有效。
系统级 PATH：对整台电脑所有用户有效。
```

不同用户、不同终端会话看到的环境变量可能不同。

### 安装了软件，不代表 Shell 一定能找到它

软件安装后，可执行文件可能真实存在，但如果 PATH 没有包含它所在的目录，命令行仍然可能提示找不到。

## 四、命令不存在时的基础检查记录

遇到命令不存在时，优先按这个顺序排查：

```text
1. 命令名是否拼错。
2. 软件是否已经安装。
3. 可执行文件实际在哪个目录。
4. PATH 是否包含可执行文件所在目录。
5. 是否重新打开终端读取了最新配置。
6. 用 Get-Command 命令名 或 where.exe 命令名 验证。
```

## 五、今日练习结论

```text
PATH 负责告诉 Shell 去哪些目录找外部命令。JAVA_HOME 通常只是指向 JDK 安装目录。命令行能不能找到 java.exe，关键要看 PATH 等查找规则是否能定位到它。
```
