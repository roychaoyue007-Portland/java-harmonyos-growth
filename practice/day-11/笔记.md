# Day 11 今日笔记：环境变量、PATH、作用范围与命令查找

日期：2026-06-28  
周内位置：Week 02 · Day 04  
主题：环境变量、PATH、作用范围、配置与命令查找  

## 1. 今天的核心问题

```text
为什么同一条命令在一台电脑上能运行，换一台电脑却可能提示找不到？
```

今天要理解的是：命令行输入一个外部命令时，Shell 需要知道这个命令对应的可执行文件在哪里。安装软件只是把文件放进电脑，PATH 等环境变量决定 Shell 是否能顺利找到它。

## 2. 环境变量是什么？

环境变量是操作系统提供给程序或 Shell 读取的键值对配置。

它的形式通常可以理解为：

```text
变量名 = 变量值
```

例子：

```text
PATH = 一串目录列表
JAVA_HOME = JDK 的安装目录
```

一句话记忆：

```text
环境变量是系统提供给程序看的公共配置信息。
```

## 3. PATH 是什么？

PATH 是一个非常重要的环境变量。

它的值通常是一串目录，Windows 中多个目录之间常用英文分号 `;` 分隔。

PATH 解决的问题是：

```text
当我只输入 java、git、node 这类命令名时，Shell 应该去哪些目录里找对应的可执行文件？
```

一句话记忆：

```text
PATH 是 Shell 查找外部命令的位置清单。
```

## 4. 变量值是什么？

变量值就是环境变量里真正保存的内容。

例如：

```text
变量名：PATH
变量值：C:\Windows\System32;C:\Program Files\Git\cmd;...
```

对 PATH 来说，变量值不是一个程序，而是一串目录列表。

## 5. 作用范围是什么？

作用范围表示这个环境变量对谁有效。

常见范围：

```text
用户级：只对当前用户有效。
系统级：对这台电脑上的所有用户有效。
当前进程级：只对当前打开的终端或程序有效。
```

用户级 PATH 和系统级 PATH 可能不同，所以同一台电脑上不同用户看到的命令查找结果也可能不同。

## 6. 命令查找是什么？

命令查找是 Shell 根据命令名寻找实际可执行文件的过程。

例如输入：

```powershell
java
```

PowerShell 会尝试找到对应的 `java.exe`。如果它不是内置命令，就会参考 PATH 中列出的目录，逐个查找是否存在 `java.exe`。

一句话记忆：

```text
命令查找就是 Shell 把命令名对应到真正可执行文件的过程。
```

## 7. PATH 和 JAVA_HOME 有什么区别？

```text
JAVA_HOME：通常指向 JDK 的安装目录。
PATH：告诉 Shell 去哪些目录里找可执行文件。
```

例子：

```text
JAVA_HOME 可能是：C:\Program Files\Java\jdk-xx
PATH 里通常需要包含：C:\Program Files\Java\jdk-xx\bin
```

所以：

```text
JAVA_HOME 指向“Java 安装在哪里”。
PATH 负责“命令行能不能直接找到 java.exe”。
```

## 8. 为什么安装了软件，不代表 Shell 一定能找到它？

因为安装软件只是把文件放到电脑硬盘上。

Shell 要能通过命令名找到它，还需要满足至少一个条件：

- 当前目录或命令路径能直接找到它。
- PATH 中包含可执行文件所在目录。
- 命令名没有拼错。
- 重新打开终端后读取到了最新环境变量。

如果软件已经安装，但 PATH 没有包含它的可执行文件目录，输入命令时仍然可能提示找不到。

## 9. 今日命令拆解

### `$env:PATH`

查看当前 PowerShell 会话中的 PATH 值。

重点观察：它是一串很长的目录列表。

### `$env:JAVA_HOME`

查看当前 PowerShell 会话中的 JAVA_HOME 值。

重点观察：它可能有值，也可能为空。为空不一定代表 Java 不能运行，但说明当前会话没有这个变量值。

### `[Environment]::GetEnvironmentVariable('PATH', 'User')`

查看用户级 PATH。

重点观察：它只属于当前用户配置。

### `[Environment]::GetEnvironmentVariable('PATH', 'Machine')`

查看系统级 PATH。

重点观察：它属于整台电脑配置，对所有用户生效。

### `Get-Command java`

让 PowerShell 告诉我它实际识别到的 `java` 命令信息。

重点观察：命令类型、名称、路径等。

### `where.exe java`

让 Windows 按命令查找规则列出找到的 `java.exe` 路径。

重点观察：具体找到的是哪个 `java.exe` 文件。

## 10. 命令不存在时的基础检查流程

遇到“命令不存在”或“无法识别命令”时，可以按顺序检查：

```text
1. 命令名有没有拼错。
2. 软件是否真的已经安装。
3. 可执行文件实际安装在哪里。
4. PATH 是否包含可执行文件所在目录。
5. 是否需要重新打开终端读取最新环境变量。
6. 用 Get-Command 或 where.exe 验证 Shell 实际能不能找到。
```

## 11. 今日重点观察

- PATH 本身就是一个环境变量。
- PATH 的值由多个目录组成。
- 用户级与系统级配置可能不同。
- 安装了软件，不代表 Shell 一定能找到它。
- `Get-Command java` 和 `where.exe java` 都能帮助观察命令查找结果。
