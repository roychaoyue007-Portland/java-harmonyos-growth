## 引导题 1：`java -version` 执行过程

原回答：

```text
1、先去查找java程序是否存在
2、报错，找不到java命令
3、调用java的获取版本的指令
4、powershell内
5、java环境是否存在
```

评分：7 / 10

整理：

```text
1. PowerShell 先解析命令：命令名是 java，参数是 -version。
2. 如果 java 不是内置命令，系统会根据 PATH 查找 java.exe。
3. 找到 java.exe 后，操作系统启动程序，并把 -version 参数传给它。
4. java 程序把版本信息显示到终端中，实际可能走 stderr。
5. 如果找不到 java，要检查 Java 是否安装、PATH 是否配置、终端是否重新打开。
```

需要记住：

`java -version` 失败时，不一定是 Java 程序本身坏了，也可能是 Shell 根本没有通过 PATH 找到 `java.exe`。

## 引导题 2：一条命令能否成功执行取决于什么

原回答：

```text
1、配置环境
2、识别
3、略
```

评分：6 / 10

整理：

```text
1. 是否在正确的当前目录 / 工作目录里执行。
2. 命令名能否被 Shell 或 PATH 找到。
3. 参数或操作对象是否写对。
4. 程序运行时是否具备需要的输入、文件、端口或权限。
5. 如果失败，先看错误信息，再看退出码或日志。
```

需要记住：

“配置环境”只是其中一类问题。路径、当前目录、参数、权限、端口也都会影响命令结果。

## 引导题 5：管道过滤命令解释

命令：

```powershell
Get-ChildItem | Where-Object Length -gt 0 | Select-Object Name
```

原回答：

```text
1、获取子项目，没有参数的时候就是当前目录
2、把Get-ChildItem获取的结果传递下来
3、过滤出大小大于0的结果
4、把过滤后的结果传递下来
5、筛选出名称
6、纵列的名称
```

评分：9 / 10

整理：

```text
1. Get-ChildItem 获取当前目录下的子项目，包括文件和目录。
2. 第一个 | 把 Get-ChildItem 的结果传给 Where-Object。
3. Where-Object Length -gt 0 过滤出 Length 大于 0 的项目。
4. 第二个 | 把过滤后的结果继续传给 Select-Object。
5. Select-Object Name 只保留 Name 这个属性。
6. 最终看到一列 Name，也就是符合条件的文件名 / 项目名。
```

需要记住：

“纵列的名称”这个理解方向是对的，专业表达可以说“输出一列 Name 属性值”。

## 引导题 6：进程、服务、localhost、错误信息

原回答：

```text
1、最大的区别在于前台进程是能与用户交互的，后台进程用户不感知
2、因为服务是启动后就一直存在，通过监听的方式，之后再处理通知，但用户不感知
3、本机
4、因为报错了不能解决实际问题
```

评分：8 / 10

整理：

```text
1. 前台进程有窗口或占用终端，通常能和用户交互；后台进程不一定直接展示给用户。
2. 服务是被系统管理的一类特殊后台进程，通常长期运行，监听请求或等待事件。
3. localhost 指本机，也就是当前这台电脑。
4. “报错了”无法解决实际问题，因为缺少路径、命令名、权限、端口、异常等关键线索。
```

需要记住：

`localhost:端口` 表示当前电脑上的某个服务入口。

## Day 14 验收题

### 题目 1：终端和 Shell 分别负责什么？

原回答：

> 终端负责与用户交互，接收输入等；shell负责处理用户输入

评分：9 / 10

整理：

终端负责与用户交互，接收输入、展示输出。Shell 负责解释和执行用户输入的命令。

### 题目 2：一条命令由哪两部分组成？比如 `java -version`。

原回答：

> 命令和参数

评分：10 / 10

整理：

一条命令通常由命令名和参数组成。`java -version` 中，`java` 是命令名，`-version` 是参数。

### 题目 3：当前目录为什么会影响命令执行结果？

原回答：

> 因为不同目录下的内容不一样，目录本身的路径也不一样

评分：9 / 10

整理：

当前目录会影响相对路径的解析。不同目录下的文件和子目录不同，所以同一条相对路径可能在不同目录中指向不同位置。

### 题目 4：PATH 的作用是什么？

原回答：

> PATH的作用是找到对应的值

评分：6 / 10

整理：

PATH 是一个环境变量，它的值是一串目录。Shell 查找外部命令时，会沿着这些目录寻找对应的可执行文件。

### 题目 5：管道和重定向的区别是什么？

原回答：

> 管道是传递数据、重定向是写入数据

评分：8 / 10

整理：

管道把一个命令的输出传给下一个命令继续处理。重定向改变输入或输出的位置，例如把命令结果写入文件。

### 题目 6：一个命令失败时，你的基础排查顺序是什么？

原回答：

> 先看报错信息，初步分析日志，尝试复现，深入分析日志

评分：8 / 10

整理：

```text
1. 先看完整错误信息。
2. 检查当前目录、路径、命令名、参数。
3. 如果是外部命令，检查 PATH 和安装情况。
4. 如果涉及服务，检查端口、进程和权限。
5. 必要时查看退出码和日志。
6. 复现问题，并用明确命令验证原因。
```
