# Day 14 练习答案：第二周复盘

## 1. 命令执行流程补全

整理答案：

```text
用户输入命令
↓
终端把输入交给 Shell
↓
Shell 解析出命令名和参数
↓
如果是内置命令，就直接执行
↓
如果不是内置命令，就根据 PATH 环境变量去查找可执行文件
↓
找到后启动程序，程序变成一个进程
↓
程序运行时可能读取标准输入 stdin
↓
正常结果输出到标准输出 stdout
↓
错误信息输出到标准错误 stderr
↓
如果要把上一个命令的结果交给下一个命令，用管道 |
↓
如果要把结果保存到文件，用重定向 > 或 >>，也可以用 Out-File
↓
如果失败，先看错误信息，再看退出码或日志
```

## 2. `Get-Content .\hello.txt` 排错题

原回答：

```text
1、Get-Content
2、.\hello.txt
3、当前目录有关
4、Get -Localtion
```

整理答案：

```text
1. 命令名是 Get-Content。
2. 参数 / 操作对象是 .\hello.txt。
3. 报找不到路径时，更可能和当前目录 / 文件路径有关，不是 PATH。
4. 应先用 Get-Location 确认当前目录。
```

需要记住：

```powershell
Get-Location
```

`Get-Location` 中间不能拆开，`Location` 的拼写也要注意。

## 3. 管道链路解释

命令：

```powershell
Get-ChildItem | Where-Object Length -gt 0 | Select-Object Name
```

原回答：

```text
1、获取子项目，没有参数的时候就是当前目录
2、把Get-ChildItem获取的结果传递下来
3、过滤出大小大于0的结果
4、把过滤后的结果传递下来
5、筛选出名称
6、纵列的名称
```

整理答案：

```text
1. Get-ChildItem 获取当前目录下的子项目，包括文件和目录。
2. 第一个 | 把 Get-ChildItem 的结果传给 Where-Object。
3. Where-Object Length -gt 0 过滤出 Length 大于 0 的项目。
4. 第二个 | 把过滤后的结果继续传给 Select-Object。
5. Select-Object Name 只保留 Name 这个属性。
6. 最终会看到一列 Name，也就是符合条件的文件名 / 项目名。
```

补充：`Length` 主要对文件有意义，目录的 Length 通常不按文件大小那样理解。

## 4. 进程、服务、localhost 与错误信息

原回答：

```text
1、最大的区别在于前台进程是能与用户交互的，后台进程用户不感知
2、因为服务是启动后就一直存在，通过监听的方式，之后再处理通知，但用户不感知
3、本机
4、因为报错了不能解决实际问题
```

整理答案：

```text
1. 前台进程通常有窗口或占用当前终端，能和用户交互；后台进程不一定直接展示给用户，但仍然在系统里运行。
2. 服务是被系统管理的一种特殊后台进程，通常长期运行，监听请求或等待事件，收到请求后再处理任务。
3. localhost 指本机，也就是当前这台电脑，常见地址是 127.0.0.1。
4. “报错了”没有足够线索。具体错误信息里通常包含失败位置、原因、路径、权限、端口、命令名等排查线索。
```

## 5. Day 14 验收题整理

### 题目 1：终端和 Shell 分别负责什么？

原回答：

> 终端负责与用户交互，接收输入等；shell负责处理用户输入

整理：

终端负责与用户交互：接收输入、显示输出。Shell 负责解释和执行用户输入的命令。

### 题目 2：一条命令由哪两部分组成？比如 `java -version`。

原回答：

> 命令和参数

整理：

一条命令通常由命令名和参数组成。`java -version` 中，`java` 是命令名，`-version` 是参数。

### 题目 3：当前目录为什么会影响命令执行结果？

原回答：

> 因为不同目录下的内容不一样，目录本身的路径也不一样

整理：

当前目录会影响相对路径的解析。不同目录下的文件和子目录不同，所以同一条相对路径在不同当前目录中可能指向不同位置，甚至找不到目标。

### 题目 4：PATH 的作用是什么？

原回答：

> PATH的作用是找到对应的值

整理：

PATH 是一个环境变量，它的值是一串目录。Shell 查找外部命令时，会沿着这些目录寻找对应的可执行文件，例如 `java.exe`、`git.exe`。

### 题目 5：管道和重定向的区别是什么？

原回答：

> 管道是传递数据、重定向是写入数据

整理：

管道用于把一个命令的输出传给下一个命令继续处理。重定向用于改变输入或输出的位置，例如把命令结果写入文件。

### 题目 6：一个命令失败时，你的基础排查顺序是什么？

原回答：

> 先看报错信息，初步分析日志，尝试复现，深入分析日志

整理：

基础排查顺序：

```text
1. 先看完整错误信息。
2. 确认当前目录、路径、命令名、参数是否正确。
3. 如果是外部命令，检查 PATH 和安装情况。
4. 如果涉及服务，检查端口、进程和权限。
5. 必要时查看退出码和日志。
6. 尝试复现问题，并用明确命令验证原因。
```

## 6. 验收结论

Day 14 验收通过。

本周主流程已经能串起来：

```text
终端输入
→ Shell 解析
→ 当前目录影响相对路径
→ PATH 查找外部命令
→ 程序运行成为进程
→ stdin / stdout / stderr 处理输入输出
→ 管道传递结果
→ 重定向保存结果
→ 失败时根据错误信息、退出码、日志和环境条件排查
```
