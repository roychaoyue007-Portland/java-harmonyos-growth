# Day 14 练习笔记：第二周命令行操作与基础排错复盘

## 1. 闭卷回忆

### Day 08：终端、Shell、PowerShell、命令与参数

终端负责和用户交互，接收输入、展示输出。Shell 负责解析用户输入的命令并执行。

命令通常由命令名和参数组成，例如：

```powershell
java -version
```

其中 `java` 是命令名，`-version` 是参数。

### Day 09：路径与当前目录

当前目录会影响相对路径的含义。不同目录下的文件不同，所以同一条相对路径在不同位置执行，可能得到不同结果。

```powershell
Get-Location
Get-Content .\hello.txt
```

如果当前目录下没有 `hello.txt`，就会提示找不到路径。

### Day 10：管道与重定向

管道用于传递数据：

```powershell
Get-ChildItem | Select-Object Name
```

重定向用于改变输出去向：

```powershell
Get-ChildItem > files.txt
```

### Day 11：PATH 与命令查找

PATH 是一个环境变量，值是一串目录。Shell 查找外部命令时，会根据 PATH 去这些目录中找对应的可执行文件。

```powershell
Get-Command java
where.exe java
```

### Day 12：进程、服务、监听与 localhost

前台进程通常有窗口或占用终端；后台进程不直接展示给用户。服务是由系统管理的特殊后台进程，通常长期运行、监听请求或等待事件。

`localhost` 表示本机，也就是当前这台电脑。

### Day 13：错误信息、退出码、日志和排错

命令失败时，不能只说“报错了”。完整错误信息里往往有路径、命令名、权限、端口、类名、异常名等关键线索。

基础排错顺序：

```text
看完整错误信息
→ 确认当前目录和路径
→ 确认命令是否能找到
→ 检查参数、权限、端口等条件
→ 必要时看退出码和日志
→ 复现并验证原因
```

## 2. 命令执行流程图

```text
用户输入命令
↓
终端把输入交给 Shell
↓
Shell 解析出命令名和参数
↓
如果是内置命令，就直接执行
↓
如果不是内置命令，就根据 PATH 环境变量查找可执行文件
↓
找到后启动程序，程序变成一个进程
↓
程序运行时可能读取标准输入 stdin
↓
正常结果输出到标准输出 stdout
↓
错误信息输出到标准错误 stderr
↓
如果要把上一个命令的结果交给下一个命令，用管道 |
↓
如果要把结果保存到文件，用重定向 >、>> 或 Out-File
↓
如果失败，先看错误信息，再看退出码或日志
```

## 3. 综合练习解释

### 练习 1

```powershell
Get-Content .\hello.txt
```

拆解：

- 命令名：`Get-Content`
- 操作对象：`.\hello.txt`
- 如果找不到路径，更可能和当前目录或文件路径有关，不是 PATH。
- 先用 `Get-Location` 确认当前目录。

### 练习 2

```powershell
Get-ChildItem | Where-Object Length -gt 0 | Select-Object Name
```

解释：

- `Get-ChildItem` 获取当前目录下的子项目。
- 第一个管道把结果交给 `Where-Object`。
- `Where-Object Length -gt 0` 过滤出 Length 大于 0 的项目。
- 第二个管道把过滤结果交给 `Select-Object`。
- `Select-Object Name` 只保留 Name 属性。
- 最终看到一列符合条件的名称。

### 练习 3

```powershell
Get-ChildItem > files.txt
Get-Content .\files.txt
```

解释：

- 第一条命令把当前目录列表写入 `files.txt`。
- 第二条命令从 `files.txt` 里读取内容。
- 这里用的是重定向，不是管道。

## 4. 排错记录模板

以后遇到命令失败，可以按这个模板记录：

```text
我执行的命令：
当前目录：
完整错误信息：
错误关键词：
最可能原因：
验证命令：
验证结果：
下一步：
```

示例：

```text
我执行的命令：Get-Content .\hello.txt
当前目录：通过 Get-Location 查看
完整错误信息：找不到路径 .\hello.txt
错误关键词：路径、not exist、hello.txt
最可能原因：当前目录下没有 hello.txt
验证命令：Test-Path .\hello.txt
验证结果：False
下一步：切换到正确目录，或使用正确路径
```

## 5. 今日复盘

今天把第二周内容串成了一条主线：从输入命令、Shell 解析、PATH 查找、进程运行、输入输出、管道重定向，到失败后的错误信息、退出码、日志和验证。

最需要继续巩固的是：

- PATH 负责找外部命令。
- 当前目录负责影响相对路径。
- 管道是传给下一个命令，重定向是写到文件或改变输入输出位置。
- 排错要先看完整错误信息，不要只说“报错了”。
