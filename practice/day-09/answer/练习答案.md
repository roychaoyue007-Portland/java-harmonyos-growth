## 一、今日笔记题答案

### 1. 文件系统是什么？

文件系统是操作系统用来组织、保存、查找和管理文件的一套结构和规则。

它解决的问题包括：

- 文件存放在哪里。
- 文件怎样被找到。
- 文件夹之间怎样形成层级关系。
- 程序怎样通过路径访问文件。

### 2. 文件和文件夹有什么区别？

文件是存储数据的基本单位，例如图片、Word 文档、文本文件、代码文件。

文件夹，也叫目录，是用来存放文件或其他文件夹的位置。

简单说：

```text
文件：保存具体内容。
文件夹：用来组织和存放文件。
```

### 3. 路径是什么？它解决什么问题？

路径是文件或文件夹的位置描述，也可以理解成文件或文件夹的地址。

它解决的问题是：

```text
我想找的文件或文件夹到底在哪里？
```

### 4. 绝对路径是什么？

绝对路径是从根位置开始写出的完整路径。

Windows 示例：

```text
C:\Codex
C:\study\day09-path-demo\notes\today.txt
```

绝对路径的特点是：不依赖当前目录，起点固定。

### 5. 相对路径是什么？

相对路径是从当前目录出发描述目标位置的路径。

示例：

```text
.\notes\today.txt
..\day08
```

相对路径的特点是：依赖当前目录。当前目录变了，同一条相对路径可能指向不同位置。

### 6. 当前目录是什么？

当前目录是 Shell 当前所在的位置。

在 PowerShell 中可以用下面命令查看：

```powershell
Get-Location
```

当前目录是相对路径的起点。

### 7. 为什么当前目录会影响相对路径？

因为相对路径不是从固定根位置开始，而是从当前目录开始。

如果当前目录变了，相对路径的出发点也变了，所以同一条相对路径可能找到不同的文件或文件夹。

### 8. `.` 表示什么？

`.` 表示当前目录。

例如：

```text
.\notes
```

意思是：从当前目录出发，进入 `notes` 文件夹。

### 9. `..` 表示什么？

`..` 表示当前目录的父目录，也就是上一级目录。

例如：

```text
..\day08
```

意思是：先回到父目录，再进入 `day08`。

### 10. `Set-Location .\notes` 和 `Set-Location ..` 分别做了什么？

```powershell
Set-Location .\notes
```

意思是：从当前目录出发，进入当前目录下的 `notes` 文件夹。

```powershell
Set-Location ..
```

意思是：回到当前目录的父目录。

### 11. 为什么同一个文件既可以用绝对路径表示，也可以用相对路径表示？

因为绝对路径和相对路径只是两种不同的定位方式。

绝对路径从根位置开始描述。

相对路径从当前目录开始描述。

如果当前目录合适，相对路径也可以定位到和绝对路径同一个文件。

例如当前目录是：

```text
C:\study\day09-path-demo
```

那么：

```text
.\notes\today.txt
```

和：

```text
C:\study\day09-path-demo\notes\today.txt
```

可以指向同一个文件。

## 二、PowerShell 练习答案

### 任务 A：观察当前目录

命令：

```powershell
Get-Location
```

答案模板：

```text
当前目录是：PowerShell 输出的 Path
这个路径是绝对路径还是相对路径：绝对路径
为什么：因为它从盘符或根位置开始，例如 C:\...
```

### 任务 B：查看当前目录内容

命令：

```powershell
Get-ChildItem
```

答案：

`Get-ChildItem` 会列出当前目录中的文件和文件夹。

常见观察方式：

- `Mode` 中以 `d` 开头的一般是目录。
- 有文件大小 `Length` 的一般是文件。
- `Name` 显示文件或文件夹名称。

### 任务 C：创建观察用目录结构

命令：

```powershell
New-Item -ItemType Directory -Name day09-path-demo
Set-Location .\day09-path-demo
New-Item -ItemType Directory -Name notes
New-Item -ItemType Directory -Name practice
New-Item -ItemType File -Name readme.txt
New-Item -ItemType File -Path .\notes\today.txt
```

完成后目录结构大致是：

```text
day09-path-demo
├─ notes
│  └─ today.txt
├─ practice
└─ readme.txt
```

### 任务 D：观察 `.`

命令：

```powershell
Get-Location
Get-ChildItem .
```

答案：

```text
. 表示当前目录。
Get-ChildItem 和 Get-ChildItem . 的结果通常类似。
因为不写路径时，Get-ChildItem 默认查看当前目录；写 . 也是查看当前目录。
```

### 任务 E：观察 `..`

命令：

```powershell
Get-Location
Get-ChildItem ..
```

答案：

```text
.. 表示父目录。
它指向当前目录的上一级目录。
它不是随便一个上级，而是当前目录结构中直接包含当前目录的那个目录。
```

### 任务 F：切换目录

命令：

```powershell
Set-Location .\notes
Get-Location
Get-ChildItem
Set-Location ..
Get-Location
```

答案：

```text
进入 notes 后，当前目录变成 day09-path-demo\notes。
执行 Set-Location .. 后，回到 day09-path-demo。
当前目录变化后，相对路径的起点也变化。
```

### 任务 G：比较绝对路径和相对路径

相对路径：

```text
.\notes\today.txt
```

绝对路径示例：

```text
C:\study\day09-path-demo\notes\today.txt
```

答案：

```text
C:\study\day09-path-demo\notes\today.txt 是绝对路径。
.\notes\today.txt 是相对路径。
如果当前目录不在 day09-path-demo，.\notes\today.txt 不一定还能找到同一个文件。
原因是 . 表示当前目录，当前目录变了，相对路径的起点也变了。
```


### 任务 H：使用 `Resolve-Path` 把相对路径变成完整路径

命令：

```powershell
Resolve-Path .
Resolve-Path ..
Resolve-Path .
otes	oday.txt
```

答案：

```text
Resolve-Path . 会显示当前目录的完整路径。
Resolve-Path .. 会显示父目录的完整路径。
Resolve-Path .
otes	oday.txt 会把这条相对路径解析成完整路径。
```

注意：如果目标路径不存在，`Resolve-Path` 会报错。这说明它不只是“拼字符串”，还会检查路径是否真实存在。

### 任务 I：区分路径存在和路径可访问

答案：

```text
路径存在：文件或文件夹确实在文件系统里。
路径可访问：当前用户或程序有权限读取、进入或操作它。
```

例子：

```text
某个系统文件夹可能存在，但普通用户没有权限修改。
某个项目目录可能存在，但如果权限不足，程序仍然无法读取。
```

排查顺序：

```text
先确认路径是否存在。
再确认当前用户或程序是否有权限访问。
```
