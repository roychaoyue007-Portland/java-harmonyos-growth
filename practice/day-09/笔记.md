# Day 09 今日笔记：文件系统、路径、当前目录

日期：2026-06-25  
周内位置：Week 02 · Day 02  
主题：文件系统、绝对路径、相对路径、当前目录与父目录  

## 1. 文件系统是什么？

文件系统是操作系统用来组织、保存、查找和管理文件的一套结构和规则。

它解决的问题是：

- 文件存放在哪里。
- 文件怎样被找到。
- 文件夹之间怎样形成层级关系。
- 程序怎样通过路径访问文件。

一句话记忆：

```text
文件系统负责让文件有地方放，也能被重新找到。
```

## 2. 文件和文件夹有什么区别？

文件是存储数据的基本单位，例如图片、Word 文档、文本文件、代码文件。

文件夹，也叫目录，是用来存放文件或其他文件夹的位置。

区别：

```text
文件：保存具体内容。
文件夹：组织和存放文件。
```

## 3. 路径是什么？它解决什么问题？

路径是文件或文件夹的位置描述，也可以理解成文件或文件夹的地址。

它解决的问题是：

```text
我要找的文件或文件夹到底在哪里？
```

路径不是文件本身，而是找到文件或文件夹的方法。

## 4. 绝对路径是什么？

绝对路径是从根位置开始写出的完整路径。

Windows 示例：

```text
C:\study\day09-path-demo\notes\today.txt
```

特点：

- 从固定起点开始。
- 不依赖当前目录。
- 当前目录改变后，它仍然指向同一个位置。

## 5. 相对路径是什么？

相对路径是从当前目录出发描述目标位置的路径。

示例：

```text
.\notes\today.txt
..\day08
```

特点：

- 起点是当前目录。
- 当前目录变了，相对路径的含义可能改变。
- 写起来更短，但必须知道当前在哪里。

## 6. 当前目录是什么？

当前目录是 Shell 当前所在的位置。

在 PowerShell 中可以用下面命令查看：

```powershell
Get-Location
```

当前目录是相对路径的起点。

## 7. 为什么当前目录会影响相对路径？

因为相对路径不是从固定根位置开始，而是从当前目录开始。

如果当前目录改变，相对路径的起点也改变，所以同一条相对路径可能指向不同位置。

例子：

```text
.\notes\today.txt
```

如果当前目录是：

```text
C:\study\day09-path-demo
```

它指向：

```text
C:\study\day09-path-demo\notes\today.txt
```

如果当前目录是：

```text
C:\study\day10-demo
```

它可能指向：

```text
C:\study\day10-demo\notes\today.txt
```

## 8. `.` 表示什么？

`.` 表示当前目录。

例如：

```text
.\notes
```

意思是：

```text
从当前目录出发，进入 notes 文件夹。
```

## 9. `..` 表示什么？

`..` 表示当前目录的父目录，也就是上一级目录。

例如：

```text
..\day08
```

意思是：

```text
先回到当前目录的父目录，再进入 day08。
```

## 10. `Set-Location .\notes` 和 `Set-Location ..` 分别做了什么？

```powershell
Set-Location .\notes
```

意思是：从当前目录出发，进入当前目录下的 `notes` 文件夹。

```powershell
Set-Location ..
```

意思是：回到当前目录的父目录。

## 11. 为什么同一个文件既可以用绝对路径表示，也可以用相对路径表示？

因为绝对路径和相对路径只是两种不同的定位方式。

绝对路径从根位置开始描述。

相对路径从当前目录开始描述。

如果当前目录合适，相对路径也可以定位到和绝对路径同一个文件。

例如当前目录是：

```text
C:\study\day09-path-demo
```

那么：

```text
.\notes\today.txt
```

和：

```text
C:\study\day09-path-demo\notes\today.txt
```

可以指向同一个文件。


## 12. `Resolve-Path` 是什么？

`Resolve-Path` 是 PowerShell 中用来把路径解析成完整路径的命令。

它常用来回答这个问题：

```text
这条相对路径最终指向电脑里的哪个完整位置？
```

示例：

```powershell
Resolve-Path .
Resolve-Path ..
Resolve-Path .
otes	oday.txt
```

理解：

```text
.  会被解析成当前目录的完整路径。
.. 会被解析成父目录的完整路径。
.
otes	oday.txt 会被解析成从当前目录出发后对应文件的完整路径。
```

注意：`Resolve-Path` 通常要求目标路径真实存在。如果路径不存在，它会报错。

## 13. 路径存在和路径可访问有什么区别？

路径存在，表示这个文件或文件夹确实在文件系统里。

路径可访问，表示当前用户或当前程序有权限读取、进入或操作它。

两者不是同一件事：

```text
路径存在：东西在那里。
路径可访问：我有权限碰它。
```

例子：

```text
一个系统目录可能真实存在，但普通用户没有权限修改它。
一个文件夹可能存在，但当前程序没有权限读取里面的内容。
```

所以排查路径问题时要分两步：

```text
第一步：路径是否存在？
第二步：路径是否能被当前用户或程序访问？
```
