原回答：
> 让我们定位错误，以便解决问题

评分：8 / 10

整理：

需要记住：
错误信息是排错的第一线索。

## 题目 2：为什么不能只说“报错了”？

原回答：
> 因为报错了并不是准确的错误信息的解读

评分：8 / 10

整理：
“报错了”只是现象，不包含命令、路径、错误关键词、异常名等线索，别人无法根据它定位问题。

需要记住：
“报错了”不是错误信息，只是问题现象。

## 题目 3：退出码 0 和非 0 通常分别表示什么？

原回答：
> 0说明是正常退出，非0是异常退出

评分：10 / 10

整理：
通常 `0` 表示成功结束，非 `0` 表示失败、异常或非正常结束。

需要记住：
0 通常表示成功，非 0 通常表示失败。

## 题目 4：$LASTEXITCODE 主要用来观察哪类程序的退出状态？

`$LASTEXITCODE` 主要用来观察外部程序的退出码，比如 `java.exe`、`git.exe`、`node.exe`。例如执行 `java NotExistClass` 后，可以用 `$LASTEXITCODE` 查看外部程序结束时留下的状态。

## 题目 5：日志和终端错误信息有什么关系？

原回答：
> 日志可能包含终端错误信息，包括了用户的操作行为，终端错误信息是一个总结性的

评分：7 / 10

整理：
终端错误信息通常是当前这次失败直接显示出来的错误；日志可能记录更完整的运行过程，包括错误发生前后的上下文。

需要记住：
终端错误信息是眼前线索，日志可能有更完整的上下文。

## 题目 6：“路径不存在”和“命令不存在”有什么区别？

原回答：
> 路径不存在是针对程序的，命令不存在是shell发出的

评分：7 / 10

整理：
路径不存在：命令已经找到了，但你给它的目标路径找不到。  
命令不存在：Shell 连要执行的命令都找不到。

例子：

```powershell
Get-ChildItem .\not-exist-folder
```

这是路径不存在。

```powershell
not-a-real-command
```

这是命令不存在。

需要记住：
路径不存在是目标找不到，命令不存在是 Shell 找不到命令。

## 题目 7：java NotExistClass 失败时，为什么不能直接说“java 没装”？

原回答：
> 因为需要根据具体的报错信息来判断，有可能java已运行

评分：8 / 10

整理：
如果 `java` 命令已经被执行了，说明 Shell 至少找到了 `java.exe`。失败可能是 Java 运行时找不到 `NotExistClass`，而不是 Java 没装。

需要记住：
`java` 能启动，不等于 Java 类能找到。

## 题目 8：请写出一套你自己的基础排错流程。

原回答：
> 先根据错误信息做出判断，然后阅读日志理解操作尝试复现

评分：16 / 20

整理：
更完整的基础排错流程：

```text
1. 记录我执行的完整命令。
2. 保留完整错误信息。
3. 找出最重要的关键词：路径、命令名、类名、异常名、端口号。
4. 确认当前路径和环境。
5. 提出最可能的原因。
6. 用一条明确命令验证这个原因。
7. 如果不成立，再换下一个假设。
```

需要记住：
排错要记录、判断、验证，而不是盲目重试。
