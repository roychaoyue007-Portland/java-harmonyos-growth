## Day 01

### 词：文件

一句话解释：文件是电脑里保存内容的一个单位，比如文本、图片、代码都可以是文件。

它解决什么问题：让数据可以被保存、读取、复制和传递。

我在哪里会遇到：`.txt`、`.java`、`.jpg`、`.md`、`.json`。

相关词：文件夹、路径、扩展名、保存。

我是否亲手做过：做过。今天用 `mkdir practice\day-01\my-first-folder` 创建过文件夹。

### 词：文件夹

一句话解释：文件夹是用来组织和存放文件的容器。

它解决什么问题：让大量文件可以分类管理。

我在哪里会遇到：项目目录、下载目录、桌面文件夹。

相关词：目录、路径、子文件夹。

我是否亲手做过：做过。今天进入过 `E:\Codex\projects\java-harmonyos-growth`，也创建过 `my-first-folder` 文件夹。

### 词：路径

一句话解释：路径是文件或文件夹在电脑里的地址。

它解决什么问题：让人和程序能准确找到某个文件或文件夹。

我在哪里会遇到：`E:\Codex\projects\java-harmonyos-growth`。

相关词：绝对路径、相对路径、当前目录。

我是否亲手做过：做过。今天使用过 `E:\Codex\projects\java-harmonyos-growth` 这样的完整路径。

### 词：命令

一句话解释：命令是人在终端里输入的一句话，用来让电脑执行某个操作。

它解决什么问题：让人可以直接控制操作系统或工具。

我在哪里会遇到：PowerShell、终端、Git、Java、npm。

相关词：终端、Shell、参数、输出。

我是否亲手做过：做过。今天执行过 `pwd`、`dir`、`cd`、`mkdir`。

### 词：脚本

一句话解释：脚本是一组提前写好的命令，让电脑按顺序自动执行。

它解决什么问题：减少重复手动操作。

我在哪里会遇到：安装环境、启动项目、打包、部署、自动化任务。

相关词：命令、终端、PowerShell、自动化。

我是否亲手做过：还没有真正写脚本文件，但已经理解脚本是一组按顺序执行的命令。

## Day 02

### 词：操作系统

一句话解释：操作系统是电脑的管理系统，负责协调 CPU、内存、硬盘和程序。

它解决什么问题：让多个程序可以有序使用电脑资源，不会互相冲突。

我在哪里会遇到：Windows 桌面、任务栏、文件管理器、任务管理器、软件启动过程。

相关词：CPU、内存、硬盘、程序、进程。

我是否亲手做过：做过。今天观察了 Windows 和任务管理器，理解操作系统在管理电脑资源。

### 词：CPU

一句话解释：CPU 是电脑的计算核心，负责执行程序里的指令。

它解决什么问题：让程序真正运行起来，完成计算、判断和处理任务。

我在哪里会遇到：任务管理器里的 CPU 使用率、电脑卡顿、软件运行、游戏性能。

相关词：操作系统、内存、程序、指令、计算。

我是否亲手做过：做过。今天通过任务管理器观察了 CPU 使用情况。

### 词：内存

一句话解释：内存是电脑临时存放正在运行的程序和数据的空间。

它解决什么问题：让 CPU 可以快速读取正在使用的数据，提高程序运行速度。

我在哪里会遇到：任务管理器里的内存占用、打开多个软件后电脑变慢。

相关词：CPU、硬盘、程序、临时空间、运行。

我是否亲手做过：做过。今天观察了任务管理器里的内存占用。

### 词：硬盘

一句话解释：硬盘是长期保存文件、软件、系统和代码的存储设备。

它解决什么问题：让数据在关机后仍然可以保存，不会像内存那样被释放。

我在哪里会遇到：C 盘、D 盘、文件夹、安装软件、保存代码。

相关词：文件、路径、内存、程序、存储。

我是否亲手做过：做过。今天理解了程序没有运行时通常保存在硬盘里。

### 词：程序

一句话解释：程序是一组让电脑执行某件事的指令集合。

它解决什么问题：把人的需求变成电脑可以执行的操作。

我在哪里会遇到：微信、浏览器、VS Code、游戏、命令行工具。

相关词：操作系统、CPU、内存、硬盘、运行。

我是否亲手做过：做过。今天理解了程序从硬盘加载到内存，再由 CPU 执行的过程。
