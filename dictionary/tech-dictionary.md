## Day 01

### 词：文件

一句话解释：文件是电脑里保存内容的一个单位，比如文本、图片、代码都可以是文件。

它解决什么问题：让数据可以被保存、读取、复制和传递。

我在哪里会遇到：`.txt`、`.java`、`.jpg`、`.md`、`.json`。

相关词：文件夹、路径、扩展名、保存。

我是否亲手做过：做过。今天用 `mkdir practice\day-01\my-first-folder` 创建过文件夹。

### 词：文件夹

一句话解释：文件夹是用来组织和存放文件的容器。

它解决什么问题：让大量文件可以分类管理。

我在哪里会遇到：项目目录、下载目录、桌面文件夹。

相关词：目录、路径、子文件夹。

我是否亲手做过：做过。今天进入过 `E:\Codex\projects\java-harmonyos-growth`，也创建过 `my-first-folder` 文件夹。

### 词：路径

一句话解释：路径是文件或文件夹在电脑里的地址。

它解决什么问题：让人和程序能准确找到某个文件或文件夹。

我在哪里会遇到：`E:\Codex\projects\java-harmonyos-growth`。

相关词：绝对路径、相对路径、当前目录。

我是否亲手做过：做过。今天使用过 `E:\Codex\projects\java-harmonyos-growth` 这样的完整路径。

### 词：命令

一句话解释：命令是人在终端里输入的一句话，用来让电脑执行某个操作。

它解决什么问题：让人可以直接控制操作系统或工具。

我在哪里会遇到：PowerShell、终端、Git、Java、npm。

相关词：终端、Shell、参数、输出。

我是否亲手做过：做过。今天执行过 `pwd`、`dir`、`cd`、`mkdir`。

### 词：脚本

一句话解释：脚本是一组提前写好的命令，让电脑按顺序自动执行。

它解决什么问题：减少重复手动操作。

我在哪里会遇到：安装环境、启动项目、打包、部署、自动化任务。

相关词：命令、终端、PowerShell、自动化。

我是否亲手做过：还没有真正写脚本文件，但已经理解脚本是一组按顺序执行的命令。

## Day 02

### 词：操作系统

一句话解释：操作系统是电脑的管理系统，负责协调 CPU、内存、硬盘和程序。

它解决什么问题：让多个程序可以有序使用电脑资源，不会互相冲突。

我在哪里会遇到：Windows 桌面、任务栏、文件管理器、任务管理器、软件启动过程。

相关词：CPU、内存、硬盘、程序、进程。

我是否亲手做过：做过。今天观察了 Windows 和任务管理器，理解操作系统在管理电脑资源。

### 词：CPU

一句话解释：CPU 是电脑的计算核心，负责执行程序里的指令。

它解决什么问题：让程序真正运行起来，完成计算、判断和处理任务。

我在哪里会遇到：任务管理器里的 CPU 使用率、电脑卡顿、软件运行、游戏性能。

相关词：操作系统、内存、程序、指令、计算。

我是否亲手做过：做过。今天通过任务管理器观察了 CPU 使用情况。

### 词：内存

一句话解释：内存是电脑临时存放正在运行的程序和数据的空间。

它解决什么问题：让 CPU 可以快速读取正在使用的数据，提高程序运行速度。

我在哪里会遇到：任务管理器里的内存占用、打开多个软件后电脑变慢。

相关词：CPU、硬盘、程序、临时空间、运行。

我是否亲手做过：做过。今天观察了任务管理器里的内存占用。

### 词：硬盘

一句话解释：硬盘是长期保存文件、软件、系统和代码的存储设备。

它解决什么问题：让数据在关机后仍然可以保存，不会像内存那样被释放。

我在哪里会遇到：C 盘、D 盘、文件夹、安装软件、保存代码。

相关词：文件、路径、内存、程序、存储。

我是否亲手做过：做过。今天理解了程序没有运行时通常保存在硬盘里。

### 词：程序

一句话解释：程序是一组让电脑执行某件事的指令集合。

它解决什么问题：把人的需求变成电脑可以执行的操作。

我在哪里会遇到：微信、浏览器、VS Code、游戏、命令行工具。

相关词：操作系统、CPU、内存、硬盘、运行。

我是否亲手做过：做过。今天理解了程序从硬盘加载到内存，再由 CPU 执行的过程。

## Day 03

### 词：程序

一句话解释：程序是保存在硬盘里的软件和指令集合，用来告诉电脑要做什么。

它解决什么问题：把人想让电脑完成的任务，写成电脑可以执行的指令。

我在哪里会遇到：`.exe` 文件、浏览器、VS Code、微信、QQ、命令行工具。

相关词：文件、硬盘、指令、运行、进程。

我是否亲手做过：做过。观察过程序没有运行时保存在硬盘里，启动后会在任务管理器里出现对应进程。

### 词：进程

一句话解释：进程是程序启动后，由操作系统创建出来的运行实例。

它解决什么问题：让操作系统可以管理正在运行的程序，比如分配内存、CPU 时间和其他资源。

我在哪里会遇到：任务管理器里的“进程”页面，PowerShell 的 `Get-Process` 命令。

相关词：程序、运行、内存、CPU、任务管理器、线程。

我是否亲手做过：做过。打开软件后，在任务管理器中观察过相关进程。

### 词：线程

一句话解释：线程是进程内部执行任务的基本单位。

它解决什么问题：让一个进程内部可以执行一个或多个任务。

我在哪里会遇到：浏览器、编辑器、聊天软件、下载任务、后台处理任务。

相关词：进程、CPU、执行、任务、调度。

我是否亲手做过：初步观察过。知道一个进程里可以有多个线程，但还没有深入查看线程细节。

### 词：任务管理器

一句话解释：任务管理器是 Windows 用来查看和管理进程、CPU、内存、磁盘等资源的工具。

它解决什么问题：帮助我看清楚电脑当前有哪些程序正在运行、哪些程序占用资源多，以及哪个程序可能卡住了。

我在哪里会遇到：按 `Ctrl + Shift + Esc` 打开任务管理器。

相关词：进程、CPU、内存、磁盘、结束任务、未响应。

我是否亲手做过：做过。用任务管理器观察过内存占用，也知道可以结束卡死或未响应的程序。

### 词：运行

一句话解释：运行是指程序被启动后，变成进程，并由线程开始执行任务的状态。

它解决什么问题：让硬盘里的静态程序真正开始工作。

我在哪里会遇到：双击软件、在命令行输入命令、打开浏览器、运行代码。

相关词：程序、进程、线程、CPU、内存、操作系统。

我是否亲手做过：做过。打开软件时，程序从硬盘里的文件变成任务管理器里的进程，这就是运行。

## Day 04

### 词：安装

一句话解释：安装是把软件需要的文件放到电脑硬盘上。

它解决什么问题：让软件真正存在于电脑里。

我在哪里会遇到：安装 JDK、Git、VS Code、浏览器、鸿蒙开发工具。

相关词：软件、硬盘、安装目录、卸载。

我是否亲手做过：做过。今天理解了安装只是让软件存在，不代表命令行一定能找到它。

### 词：配置

一句话解释：配置是告诉系统或软件应该怎么工作。

它解决什么问题：让软件按照正确的规则被使用。

我在哪里会遇到：配置 PATH、配置 Git 用户名、配置 IDE、配置项目参数。

相关词：设置、环境变量、PATH、配置文件。

我是否亲手做过：做过。今天理解了配置和安装不是一回事。

### 词：环境变量

一句话解释：环境变量是操作系统保存的全局配置信息。

它解决什么问题：让系统和多个程序共享一些公共信息。

我在哪里会遇到：Windows 系统环境变量、PATH、JAVA_HOME、TEMP。

相关词：操作系统、配置、PATH、JAVA_HOME。

我是否亲手做过：初步观察过。知道它属于系统层面的公共配置。

### 词：PATH

一句话解释：PATH 是告诉系统输入命令时去哪些文件夹里找程序的环境变量。

它解决什么问题：让 PowerShell 可以直接识别 `java`、`git` 等命令。

我在哪里会遇到：PowerShell、JDK 配置、Git 配置、Node.js 配置。

相关词：环境变量、命令、可执行文件、目录。

我是否亲手做过：观察过。知道可以用 `where.exe java` 或 `where.exe git` 查看系统实际找到的路径。

### 词：版本号

一句话解释：版本号是软件当前这一版的编号。

它解决什么问题：帮助确认软件功能、兼容性和排查问题范围。

我在哪里会遇到：`java -version`、`git --version`、软件关于页面、项目文档。

相关词：软件、更新、兼容性、命令。

我是否亲手做过：做过。知道版本号不是软件名字，而是软件当前这一版的编号。

## Day 05

### 词：源代码

一句话解释：源代码是程序员用编程语言写出来、保存在文件里的代码。

它解决什么问题：让人可以用相对容易理解的语言描述程序逻辑。

我在哪里会遇到：`.java`、`.js`、`.py`、`.ets` 等代码文件。

相关词：文件、编译、运行、程序、语法。

我是否亲手做过：做过。今天理解了 `.java` 文件只是源代码文件，不等于程序已经运行。

### 词：编译

一句话解释：编译是把源代码转换成更适合执行的形式。

它解决什么问题：让人写的代码变成运行环境更容易理解和执行的内容。

我在哪里会遇到：`javac Hello.java`、项目构建、IDE 编译。

相关词：源代码、字节码、编译器、运行。

我是否亲手做过：初步做过。知道 Java 中 `javac` 会把 `.java` 编译成 `.class`。

### 词：运行

一句话解释：运行是让程序真正开始执行任务。

它解决什么问题：让静态的代码或程序文件变成正在工作的状态。

我在哪里会遇到：`java Hello`、点击软件图标、启动项目。

相关词：程序、进程、线程、运行时、操作系统。

我是否亲手做过：做过。知道 `java Hello` 会启动 JVM 并执行程序。

### 词：解释执行

一句话解释：解释执行是运行时一边读取代码或中间表示，一边执行。

它解决什么问题：让程序不必先完整编译成最终目标文件，也能被逐步执行。

我在哪里会遇到：脚本语言、命令行、某些运行时环境。

相关词：编译、运行、解释器、运行时。

我是否亲手做过：理解了基本概念，但还需要以后结合具体语言继续观察。

### 词：运行时

一句话解释：运行时是程序运行时依赖的环境和支持系统。

它解决什么问题：为程序执行提供加载、内存管理、错误处理和基础能力。

我在哪里会遇到：Java 的 JVM、Node.js、浏览器 JavaScript 引擎。

相关词：JVM、运行、字节码、内存、程序。

我是否亲手做过：今天重点补了 Java 里的 JVM 运行时概念。

## Day 06

### 词：客户端

一句话解释：客户端是与用户交互、主动向服务端发送请求的一方。

它解决什么问题：解决用户如何使用软件、如何发起操作的问题。

我在哪里会遇到：浏览器、鸿蒙 App、手机 App、桌面软件。

相关词：服务端、请求、响应、HTTP、页面。

我是否亲手做过：做过。今天能用浏览器和鸿蒙 App 作为例子理解客户端。

### 词：服务端

一句话解释：服务端是接收客户端请求、处理数据并返回结果的一方。

它解决什么问题：解决后台处理业务、保存数据、返回结果的问题。

我在哪里会遇到：Java 后端、登录服务、订单服务、接口服务。

相关词：客户端、服务器、接口、数据库、HTTP。

我是否亲手做过：初步理解过。知道 Java 后端将来会作为服务端处理客户端请求。

### 词：IP

一句话解释：IP 地址是网络中用来标识一台设备的地址。

它解决什么问题：解决客户端要访问哪一台机器的问题。

我在哪里会遇到：访问网站、连接服务器、局域网设备通信。

相关词：服务器、网络、地址、域名、客户端。

我是否亲手做过：初步观察过。知道客户端可以通过 IP 找到服务端所在的机器。

### 词：端口

一句话解释：端口是用来区分一台机器上不同服务的编号。

它解决什么问题：解决找到机器后，应该访问哪个具体服务的问题。

我在哪里会遇到：Web 服务端口 80、HTTPS 端口 443、Java 后端常见端口 8080、MySQL 端口 3306。

相关词：IP、服务、服务器、HTTP、后端。

我是否亲手做过：初步理解过。知道同一台服务器可以运行多个服务，端口用来区分具体服务。

### 词：HTTP

一句话解释：HTTP 是客户端和服务端之间常用的通信协议。

它解决什么问题：解决客户端和服务端按照什么规则发送和接收消息的问题。

我在哪里会遇到：浏览器访问网页、App 请求接口、前端调用后端接口。

相关词：客户端、服务端、请求、响应、状态码、接口。

我是否亲手做过：初步观察过。知道 HTTP 让客户端和服务端按共同规则交流。

## Day 08

### 词：终端（Terminal）

一句话解释：终端是接收用户文字输入并显示命令结果的交互界面。

它解决什么问题：让用户能够通过文字与 Shell 交互，并看到正常结果或错误信息。

我在哪里会遇到：Windows Terminal、Codex 的终端面板、IDE 内置终端。

相关词：Shell、PowerShell、命令、输入、输出。

我是否亲手做过：做过。今天在终端中输入 PowerShell 和 Java 命令，并观察了返回结果。

### 词：Shell

一句话解释：Shell 是读取、解析并执行用户命令的程序。

它解决什么问题：把用户输入的文字转换成对 Cmdlet、脚本或外部程序的调用。

我在哪里会遇到：PowerShell、Command Prompt、Bash。

相关词：终端、命令、参数、PowerShell、外部程序。

我是否亲手做过：做过。今天能够说明终端负责输入和显示，Shell 负责读取、解析和执行。

### 词：PowerShell

一句话解释：PowerShell 是 Windows 上常用的一种 Shell 和脚本环境。

它解决什么问题：用于执行命令、操作文件、调查系统信息和运行脚本。

我在哪里会遇到：`PS>` 提示符、Windows Terminal、Codex 终端、自动化脚本。

相关词：Shell、Cmdlet、终端、脚本、命令。

我是否亲手做过：做过。今天使用过 `Get-ChildItem`、`Get-Command` 和 `Get-Help`。

### 词：命令（Command）

一句话解释：命令是告诉 Shell 本次要执行什么动作的名称。

它解决什么问题：让用户能够明确要求电脑执行某项操作。

我在哪里会遇到：`Get-ChildItem`、`Get-Command`、`java`、`javac`。

相关词：Shell、参数、Cmdlet、外部程序、PATH。

我是否亲手做过：做过。今天调查并执行了 PowerShell Cmdlet 和 Java 外部命令。

### 词：参数（Parameter / Argument）

一句话解释：参数是跟在命令后面的附加信息，用来说明对谁操作或怎样操作。

它解决什么问题：让同一个命令可以针对不同对象、范围和执行方式工作。

我在哪里会遇到：`Get-ChildItem -Path . -File` 中，`-Path` 是参数名，`.` 是参数值，`-File` 是不需要参数值的开关参数。

相关词：命令、参数名、参数值、开关参数、Shell。

我是否亲手做过：做过。今天已经能够区分参数名、参数值和开关参数。
