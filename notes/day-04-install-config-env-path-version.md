## 阶段 2：核心关系，30 分钟

今天要先记住这条关系：

```text
安装：让软件存在于硬盘上
配置：告诉系统或软件怎么使用
环境变量：操作系统保存的全局配置
PATH：环境变量中负责查找命令的一项
版本号：软件当前这一版的编号
```

安装 JDK、Git 以后，软件文件确实已经在电脑里。

但是 PowerShell 输入 `java`、`javac`、`git` 时，Windows 还需要知道这些命令对应的程序在哪里。

如果对应目录没有加入 PATH，软件虽然安装了，命令行仍然可能找不到它。

## 阶段 3：动手观察，60 分钟

在 PowerShell 中观察：

```powershell
java -version
javac -version
git --version
where.exe java
where.exe javac
where.exe git
```

观察重点：

- `java -version`：观察当前能运行的 Java 版本。
- `javac -version`：观察当前能运行的 Java 编译器版本。
- `git --version`：观察当前能运行的 Git 版本。
- `where.exe java`：观察系统实际找到了哪个路径下的 `java.exe`。
- `where.exe javac`：观察系统实际找到了哪个路径下的 `javac.exe`。
- `where.exe git`：观察系统实际找到了哪个路径下的 `git.exe`。

如果提示命令无法识别，可能说明：

1. 软件没有安装。
2. 软件安装了，但命令目录没有加入 PATH。
3. PATH 路径写错了。
4. 修改 PATH 后，没有重新打开 PowerShell。
5. 命令名字输错了。

## 我的验收答案整理

1. 安装一个软件，就是让软件安装在电脑硬盘上。修正后理解：安装是把软件需要的文件放到电脑硬盘上，让这个软件真实存在于电脑里。
2. 配置一个软件，就是告诉操作系统这个软件能干什么、怎么用它、它在哪里。修正后理解：配置是告诉系统或软件应该怎么工作、去哪里找文件、按什么规则使用。
3. 安装是让应用存在于硬盘上，静止地存在；配置告诉系统怎么运用它。修正后理解：安装偏向“软件存在”，配置偏向“如何使用”。
4. 环境变量我一开始不知道，只知道它是全局的。修正后理解：环境变量是操作系统保存的全局配置信息，不是某个软件私有的设置。
5. PATH 环境变量主要是为了在全局环境找到对应应用。修正后理解：PATH 告诉系统输入命令时去哪些文件夹里找程序。
6. 安装了 Git，但 PowerShell 输入 `git` 仍然可能失败，可能是没有配置环境。修正后理解：也可能是 Git 没装好、PATH 没加好、终端没重开或命令输错。
7. `java -version` 观察版本，`where.exe java` 观察路径。
8. 版本号不是应用当前的名字，而是软件当前这一版的编号。不同版本可能功能不同、兼容性不同，所以开发时要确认版本号。

### 补充：为什么安装了软件，命令行还可能找不到？

因为安装只说明软件文件已经存在于硬盘上。

命令行能不能找到它，还取决于系统是否知道应该去哪里找对应的可执行文件。

PATH 就是这条“查找路线”。如果软件的命令目录不在 PATH 里，PowerShell 输入命令时就可能找不到。

### 补充：为什么有时候要用 `where.exe`？

`where.exe` 可以显示系统实际找到的命令路径。

比如：

```powershell
where.exe java
where.exe git
```

它能帮助我确认：

- 系统是否真的找到了这个命令。
- 找到的是哪个目录下的程序。
- 电脑上是否有多个同名命令。
- 当前 PATH 是否符合预期。
