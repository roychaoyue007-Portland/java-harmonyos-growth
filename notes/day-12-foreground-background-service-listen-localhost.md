# Day 12 今日笔记：前台程序、后台程序、服务、监听与本机端口

日期：2026-06-28  
周内位置：Week 02 · Day 05  
主题：前台程序、后台程序、服务、监听与本机端口  

## 1. 今天的核心问题

```text
为什么有些程序关掉窗口就结束，而有些服务能一直在后台等待请求？
```

今天要理解的是：程序启动后通常会表现为进程。有些进程需要用户直接交互，关闭窗口或终端后就结束；有些程序适合长期在后台运行，甚至作为服务等待请求。监听端口表示某个进程正在等待连接。

## 2. 前台程序是什么？

前台程序是当前需要用户直接交互、占用窗口或终端的程序。

它不一定必须有图形窗口。在命令行里，一个正在占用终端、等待用户操作或等待自己执行完成的程序，也可以理解为前台程序。

一句话记忆：

```text
前台程序是我正在直接面对和交互的程序。
```

## 3. 后台程序是什么？

后台程序是在用户没有直接盯着它操作时，仍然可以继续运行的程序。

它可能没有明显窗口，也可能只在任务栏、托盘或系统后台运行。

一句话记忆：

```text
后台程序是在背后继续运行的程序。
```

## 4. 服务是什么？

服务是由系统或服务管理器管理、适合长期后台运行的程序。

服务通常用来持续提供某种能力，例如等待请求、同步数据、监听连接或执行系统任务。

服务不是“永远不能停止”的程序。它也可能处于：

```text
正在运行
已停止
正在启动
正在停止
```

一句话记忆：

```text
服务是被系统管理、适合长期后台运行的程序。
```

## 5. 进程和服务有什么关系？

进程是程序运行起来后的实例。

服务通常也需要由某个进程来承载或执行，只是它的管理方式和普通程序不同，更偏长期后台运行。

理解关系：

```text
程序启动后可能成为进程。
服务通常也会通过进程运行。
服务更强调“由系统管理、长期后台提供能力”。
```

## 6. 监听是什么？

监听是指某个进程打开了一个端口，并等待连接请求。

监听不表示“连接来了才启动进程”。更准确地说，通常是：

```text
进程已经在运行
    → 打开某个端口
    → 等待别人连接
    → 连接来了以后处理请求
```

一句话记忆：

```text
监听端口表示某个进程正在等待连接。
```

## 7. localhost 是什么？

`localhost` 表示当前这台电脑，也叫本地主机。

它不是公网服务器，也不是别人的电脑。

一句话记忆：

```text
localhost 就是当前这台电脑自己。
```

## 8. localhost:端口 表示什么？

`localhost:端口` 表示当前这台电脑上的某个服务入口。

例子：

```text
localhost:8080
```

可以理解为：

```text
当前电脑上的 8080 号服务窗口。
```

端口不是电脑编号，而是同一台电脑上区分不同服务的编号。

## 9. 今日命令拆解

### `Get-Process`

查看当前电脑正在运行的进程。

重点观察：进程名称、进程 ID、资源占用等。

### `Get-Service | Select-Object -First 10`

查看服务列表中的前 10 条。

重点观察：服务名称、显示名称、状态。

今天不纠结排序规则，重点是认识服务状态和服务名称。

### `Get-NetTCPConnection -State Listen | Select-Object -First 10`

查看当前正在监听的 TCP 连接记录中的前 10 条。

重点观察：本地地址、本地端口、状态、所属进程 ID。

## 10. 关闭窗口、停止进程、停止服务有什么区别？

```text
关闭窗口：可能只是关闭前台界面，也可能导致前台程序退出。
停止进程：直接结束某个正在运行的程序实例。
停止服务：通过服务管理机制停止某个服务，服务对应的监听端口通常也会消失。
```

不要随意停止系统关键进程或服务。

## 11. 今日重点观察

- 前台程序不一定等于“有窗口”，命令行里占用终端的程序也可以是前台程序。
- 服务通常由系统管理，适合长期后台运行，但它也可以停止。
- 监听端口表示某个进程已经在等待连接。
- `localhost` 表示当前这台电脑。
- `localhost:端口` 表示当前电脑上的某个服务入口。
