## 3. 命令执行总流程

### 3.1 以 `java -version` 为例

```text
PowerShell 先解析输入：
- 命令名：java
- 参数：-version

如果 java 不是 PowerShell 内置命令：
- Shell 根据 PATH 环境变量中的目录逐个查找 java.exe

找到 java.exe 后：
- 操作系统启动 java.exe
- java.exe 变成一个正在运行的进程
- -version 作为参数传给 java.exe
- java.exe 输出版本信息到终端

如果找不到 java：
- 检查 Java 是否安装
- 检查 PATH 是否配置
- 检查配置后是否重新打开终端
```

需要特别记住：`java -version` 的版本信息通常会显示在终端里，但它不一定走普通标准输出，实际可能走 stderr。

### 3.2 一条命令能否成功的常见影响因素

```text
1. 当前目录是否正确
2. 命令名是否能被 Shell 或 PATH 找到
3. 参数或操作对象是否正确
4. 需要的文件、输入、端口或权限是否存在
5. 程序运行后是否产生正常输出、错误输出或日志
```

当前目录尤其容易被忽略。比如：

```powershell
Get-Content .\hello.txt
```

如果当前目录里没有 `hello.txt`，就会报找不到路径。这不是 PATH 的问题，而是当前目录和文件路径的问题。

## 4. 管道与重定向复盘

### 4.1 管道

```powershell
Get-ChildItem | Where-Object Length -gt 0 | Select-Object Name
```

解释：

```text
Get-ChildItem：获取当前目录下的子项目。
第一个 |：把结果传给 Where-Object。
Where-Object Length -gt 0：过滤出 Length 大于 0 的项目。
第二个 |：把过滤后的结果继续传给 Select-Object。
Select-Object Name：只保留 Name 属性。
最终结果：输出一列符合条件的名称。
```

管道解决的是“上一个命令的结果怎么交给下一个命令继续处理”。

### 4.2 重定向

```powershell
Get-ChildItem > files.txt
Get-ChildItem >> files.txt
```

`>` 表示把输出写入文件，通常会覆盖原内容。  
`>>` 表示把输出追加到文件末尾。  

在 PowerShell 中也经常使用：

```powershell
Get-ChildItem | Out-File .\work\week-02-files.txt
Get-Content .\work\week-02-files.txt
```

重定向解决的是“输出结果去哪里”，比如从终端改为文件。

## 5. 进程、服务、监听与 localhost

程序启动后，通常会变成进程。

前台进程通常有窗口或占用当前终端，能直接和用户交互。关闭窗口或终端后，前台进程往往也结束。

后台进程不一定直接展示在用户面前，但仍然在系统中运行。

服务是一种特殊的后台进程：

```text
服务通常由系统管理
服务适合长期运行
服务可以监听端口或等待事件
服务收到请求后再处理任务
```

`localhost` 表示本机，也就是当前这台电脑。常见地址是 `127.0.0.1`。

当看到 `localhost:8080` 时，可以理解为：访问当前电脑上监听 8080 端口的某个服务。

## 6. 基础排错流程

命令失败时，不要直接猜，也不要只说“报错了”。推荐流程是：

```text
1. 记录完整命令
2. 复制完整错误信息
3. 提取关键词：路径、命令名、端口、权限、类名、not found、not recognized 等
4. 确认当前目录和相关路径
5. 确认命令是否能被找到，例如 Get-Command java
6. 确认参数或操作对象是否正确
7. 必要时查看退出码和日志
8. 提出一个可能原因
9. 用一条明确命令验证这个原因
10. 如果不成立，再换下一个假设
```

几个常见问题的判断：

| 现象 | 更可能的问题 | 先检查什么 |
| --- | --- | --- |
| `Get-Content .\hello.txt` 找不到路径 | 当前目录或文件路径错误 | `Get-Location`、`Test-Path .\hello.txt` |
| `java` 无法识别 | PATH 或安装问题 | `Get-Command java`、`where.exe java` |
| 程序提示端口被占用 | 已有进程监听同一端口 | `Get-NetTCPConnection -State Listen` |
| 程序启动失败但终端信息很少 | 需要更多上下文 | 查看日志、复现步骤、退出码 |

## 7. 本周薄弱点

### 7.1 PATH 的精确作用

薄弱表现：容易说成“PATH 找到对应的值”。

整理后：PATH 是一个环境变量，它的值是一串目录。Shell 查找外部命令时，会沿着这些目录寻找对应的可执行文件，例如 `java.exe`。

### 7.2 当前目录与 PATH 的区别

薄弱表现：遇到找不到文件时，容易想到环境配置。

整理后：如果命令名找不到，优先怀疑 PATH 或安装；如果命令存在，但操作对象找不到，优先怀疑当前目录、相对路径或文件是否存在。

### 7.3 重定向的表达

薄弱表现：把保存输出说成 “file 命令生成文件”。

整理后：保存输出靠的是重定向或 `Out-File`，不是 `file` 命令。

示例：

```powershell
Get-ChildItem > files.txt
Get-ChildItem | Out-File .\files.txt
```
