# Day 03 - 程序、进程、线程

日期：2026-06-17

## 今天的核心问题

程序运行起来以后，电脑里到底发生了什么？

## 核心概念

程序是保存在硬盘里的软件和指令集合，用来告诉电脑要做什么。程序没有启动时是静态的，只是文件，还没有真正开始执行。

进程是程序启动以后，由操作系统创建出来的运行实例。程序没有运行时是静态文件；程序运行起来后，就会变成进程。

线程是进程内部执行任务的基本单位。一个进程可以有一个线程，也可以有多个线程。线程负责具体执行代码、处理计算和完成任务。

## 程序、进程、线程的关系

```text
程序文件保存在硬盘里
    -> 用户双击程序或用命令启动程序
    -> 操作系统创建进程
    -> 操作系统给进程分配内存和资源
    -> 进程里的线程开始执行任务
    -> 程序进入运行状态
```

重点区别：

- 程序是静态的，通常保存在硬盘里。
- 进程是动态的，是程序运行起来后的实例。
- 线程属于进程，是进程内部执行任务的单位。
- 运行表示程序从静态文件变成正在工作的状态。

## 任务管理器观察

任务管理器可以通过 `Ctrl + Shift + Esc` 打开。

任务管理器里常见的指标：

- CPU：观察电脑当前有多少计算任务正在被处理，哪些进程比较占用计算能力。
- 内存：观察正在运行的进程占用了多少临时运行空间。
- 磁盘：观察硬盘当前是否在频繁读取或写入数据。
- 进程：观察当前有哪些程序正在运行。
- 未响应：观察某个程序是否卡住。

如果一个程序卡住了，可以在任务管理器中观察它是否显示“未响应”，也可以看它的 CPU、内存、磁盘占用是否异常。如果确认程序已经卡死，可以选中它并点击“结束任务”。

## 问题整理

### 程序和进程有什么区别？

程序是静态的文件，通常保存在硬盘里。进程是动态的，是程序启动后正在运行的实例。

### 一个程序没有运行时，通常在哪里？

一个程序没有运行时，通常作为文件保存在硬盘里。

### 一个程序运行起来后，操作系统会为它创建什么？

操作系统会为它创建进程，并为这个进程分配内存、CPU 时间和其他资源。

### 进程为什么会占用内存？

因为进程运行时需要临时空间保存代码、数据和运行状态。内存就像电脑给正在运行的任务开辟的临时工作空间。

### 线程和进程是什么关系？

线程属于进程，是进程内部执行任务的基本单位。一个进程中可以有一个或多个线程。

### 任务管理器里的 CPU、内存、磁盘分别能帮助我观察什么？

- CPU：观察当前计算任务的忙碌程度。
- 内存：观察进程占用了多少临时运行空间。
- 磁盘：观察硬盘是否正在频繁读取或写入数据。

### 为什么一个浏览器可能对应多个进程？

因为现代浏览器通常采用多进程设计。不同标签页、插件、扩展、网页渲染任务和后台服务，可能会被拆到不同进程里运行。这样可以提高稳定性和安全性。

### 如果一个程序卡住了，我可以如何用任务管理器观察并处理？

可以打开任务管理器，观察这个程序是否显示“未响应”，以及它的 CPU、内存、磁盘占用是否异常。如果确认程序已经卡住，可以右键这个进程，选择“结束任务”。

## 今日总结

程序是保存在硬盘里的静态文件和指令集合。程序启动以后，操作系统会创建进程，并为进程分配内存和资源。线程是在进程内部执行任务的基本单位，一个进程可以包含一个或多个线程。

任务管理器可以帮助我观察正在运行的进程，以及它们占用的 CPU、内存和磁盘资源。如果程序卡住了，可以通过任务管理器观察它是否未响应，并在必要时结束任务。
