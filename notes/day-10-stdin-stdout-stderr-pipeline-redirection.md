# Day 10 今日笔记：标准输入、输出、错误输出、管道与重定向

## 1. 今天的核心问题

```text
命令产生的结果去了哪里，又怎样把一个命令的结果交给另一个命令？
```

今天要理解的是：命令不只是“执行一下”，它还会接收输入、产生正常输出、产生错误输出，并且这些结果可以显示在终端、传给下一个命令，或者写入文件。

## 2. 标准输入是什么？

标准输入是命令或程序接收数据的入口。

它可以来自：

- 用户在键盘输入的内容。
- 上一个命令通过管道传来的结果。
- 文件或其他数据来源。

一句话记忆：

```text
标准输入是程序接收数据的入口。
```

## 3. 标准输出是什么？

标准输出是命令正常执行后产生的结果。

在 PowerShell 中，正常输出通常会显示在终端里，也可以继续交给管道右边的命令，或者写入文件。

示例：

```powershell
Write-Output "hello"
```

这条命令产生正常输出：

```text
hello
```

一句话记忆：

```text
标准输出是命令成功工作后交出来的正常结果。
```

## 4. 错误输出是什么？

错误输出是命令执行失败或遇到异常时产生的错误信息。

例如输入一个不存在的命令：

```powershell
not-a-real-command
```

PowerShell 会显示错误信息，提示这个命令无法识别。

错误输出和正常输出都可能显示在终端中，但含义不同：

```text
正常输出：命令成功后的结果。
错误输出：命令失败或异常时的说明。
```

## 5. 管道是什么？

管道用符号 `|` 表示，用来把左边命令的结果交给右边命令继续处理。

示例：

```powershell
Get-ChildItem | Select-Object Name
```

理解：

```text
Get-ChildItem        获取当前目录下的项目
|                    把左边结果交给右边
Select-Object Name   只选出 Name 这一列
```

一句话记忆：

```text
管道解决“把上一步结果交给下一步处理”的问题。
```

## 6. 重定向是什么？

重定向是把输出从默认位置改到另一个位置。

在今天练习中，用 `Out-File` 把结果写入文件：

```powershell
Get-ChildItem | Out-File .\work\file-list.txt
```

这条命令的结果不会主要显示在终端，而是写入：

```text
.\work\file-list.txt
```

然后可以用下面命令读取文件内容：

```powershell
Get-Content .\work\file-list.txt
```

一句话记忆：

```text
重定向解决“结果不要只显示在屏幕上，而是保存到别处”的问题。
```

## 7. 终端显示、管道传递、写入文件的区别

```text
终端显示：让我直接看到结果。
管道传递：把结果交给下一个命令处理。
写入文件：把结果保存下来，之后还能再查看。
```

这三种方式都和输出有关，但目的不同。

## 8. 今日命令拆解

### `Write-Output "hello"`

产生一条正常输出：

```text
hello
```

### `Get-ChildItem | Select-Object Name`

先获取当前目录中的项目，再只保留 `Name` 属性。

### `Get-ChildItem | Where-Object Length -GT 0`

先获取当前目录中的项目，再筛选 `Length` 大于 0 的项目。初学阶段主要把它理解为筛选有内容的文件。

### `Get-ChildItem | Out-File .\work\file-list.txt`

先获取当前目录中的项目，再把结果写入 `file-list.txt`。

### `Get-Content .\work\file-list.txt`

读取 `file-list.txt` 中保存的内容，并显示在终端里。

## 9. 今日重点观察

- `|` 左边的输出会成为右边命令的输入。
- 终端显示内容和写入文件不是同一件事。
- PowerShell 管道传递的是对象，初学阶段先理解为“把结果交给下一步”。
- 正常输出和错误输出都可能出现在终端，但代表的含义不同。
