## 阶段 2：核心关系，30 分钟

今天要先记住这条运行流程：

```text
程序平时保存在硬盘里
    -> 打开程序时，操作系统把程序加载到内存
    -> CPU 执行程序里的指令
    -> 操作系统负责管理 CPU、内存、硬盘和程序
```

几个重点区别：

- 硬盘负责长期保存。
- 内存负责临时运行。
- CPU 负责执行计算和指令。
- 操作系统负责协调和管理资源。
- 程序是一组要被电脑执行的指令集合。

## 我的验收答案整理

1. Windows 是一个操作系统，负责协调 CPU、内存等硬件，使它们有序工作。
2. CPU 主要负责实际执行指令，比如运行程序、计算和处理逻辑。
3. 内存叫“临时空间”，因为它用来临时保存正在运行的程序和数据；程序关闭或电脑关机后，这些数据通常会被释放，不像硬盘那样长期保存。
4. 硬盘是长期存储设备，断电后数据仍然可以保留，所以文件、软件、图片、代码都可以长期放在硬盘里。
5. 一个程序没有运行时，通常保存在硬盘里。
6. 一个程序运行时，会被加载到内存里。
7. 程序打开时，操作系统把程序从硬盘加载到内存，CPU 再执行程序里的指令。
8. CPU 表示当前电脑的计算负载；内存表示正在运行的程序占用了多少临时空间；磁盘表示硬盘正在进行多少读取和写入操作。

### 补充：为什么程序不能只放在硬盘里就算运行？

硬盘适合长期保存文件，但速度比内存慢。程序如果只是放在硬盘里，只能说明它被保存了，还没有真正开始工作。

当我们双击一个软件或用命令启动一个程序时，操作系统会把程序需要的内容加载到内存里。CPU 再从内存中读取指令和数据，开始执行。这个过程发生后，程序才算真正“运行起来”。

### 补充：为什么打开太多软件电脑会变慢？

每个正在运行的软件都会占用内存，也可能让 CPU 执行更多任务。

如果软件开得太多，内存可能被大量占用，CPU 也要在多个任务之间频繁切换。电脑就会变慢、卡顿，甚至需要把一部分数据临时写到硬盘里，速度会更慢。
