## 阶段 2：概念分类与易混词辨析，45 分钟

把本周词汇分成 5 组：

- 文件与操作：文件、目录、路径、命令、脚本。
- 计算机资源：操作系统、CPU、内存、硬盘。
- 程序运行：程序、进程、线程、运行、任务管理器。
- 开发环境：安装、配置、环境变量、PATH、版本号。
- 代码与网络：源代码、编译、解释执行、运行时、客户端、服务端、IP、端口、HTTP。

重点用自己的话回答这些区别：

1. 文件和目录有什么区别？
文件是存放具体内容的东西，它可以放在目录中，然后目录中也可以放目录
2. 硬盘和内存有什么区别？
硬盘是存放静态程序的位置，是存储数据的位置。内存的话是程序运行时操作系统给他分配的类似工作台的空间
3. 程序和进程有什么区别？
程序是静态的，程序打开之后以进程的形式存在于这个任务管理器中，相当于是个动态的
4. 进程和线程有什么关系？
线程是进程的最小可执行单位
5. 安装和配置有什么区别？
安装只是把程序安装在计算机上，配置的话是让这个程序可以使用
6. 环境变量和 PATH 有什么关系？
这个不知道，还没搞懂
7. 编译和运行有什么区别？
编译相当于是一个解释的过程，最终的产物，是虚拟机可以阅读的字节码。运行是让一个程序可以执行起来。程序可以运行，是建立在源码被编译成虚拟机可以阅读的内容的基础上的
8. IP 和端口分别解决什么问题？
IP负责找到服务端，端口的话，是找到服务端上的具体的服务
9. 客户端和服务端分别负责什么？
客户端负责与用户交互、存放数据等等。服务端负责处理数据等等

要求：每个问题都要包含“是什么、有什么不同、举一个例子”。

## 阶段 6：Day 07 验收测试，35～45 分钟

闭卷回答，每题 10 分：

1. 路径、命令和 PATH 是怎样配合工作的？
不知道
2. 操作系统、CPU、内存和硬盘在程序运行时分别做什么？
程序是从硬盘上启动的，操作系统给程序分配内存，程序在内存上变为动态的进程，并且进程下面有很多执行具体任务的线程，CPU执行这些线程，进而完成具体的任务
3. 程序、进程和线程之间是什么关系？
程序是静态的存放在硬盘上的，在程序运行时会变成进程存在于内存之上，然后线程是进程的最小执行单位
4. 安装、配置和版本号分别解决什么问题？
安装解决的是计算机上没有这个应用或者说程序的问题；配置是告诉计算机怎么使用这个程序；版本号是给这个程序一个特定的版本，就是表达他能够干什么
5. Java 源代码从 `.java` 文件到运行结果，大致经历哪些步骤？
大致要经过编译成字节码，虚拟机解释并且执行对应的字节码，展示对应的执行结果
