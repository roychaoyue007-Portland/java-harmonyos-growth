## 1. 今天的核心问题

命令执行失败时，我应该从哪里开始查，而不是盲目重试？

今天最重要的不是多学几个复杂命令，而是建立一个稳定的排错习惯：

```text
先保留现场
再阅读错误信息
再判断问题类型
再提出原因

```

不要只说“报错了”。真正有用的信息通常包括：

```text
我执行了什么命令
在哪个目录执行

退出码是什么
是否有日志或更多上下文
我怎样复现这个问题
```

## 2. 核心词

### 错误信息（Error Message）

错误信息是命令或程序失败时给出的说明。

它通常会提供路径、命令名、文件名、类名、异常名、端口号等线索。

一句话记忆：

```text
错误信息是失败时的第一线索。
```

### 退出码（Exit Code）

退出码是程序结束时返回给 Shell 或操作系统的数字状态。

通常：

```text
0：成功结束
非 0：失败、异常或非正常结束
```

PowerShell 中可以用 `$LASTEXITCODE` 观察外部程序的退出状态，例如 `java.exe`、`git.exe`、`node.exe`。

### 日志（Log）

日志是程序运行过程中留下的事件、状态、警告和错误记录。

终端错误信息通常是眼前这次失败直接显示出来的错误；日志可能记录错误发生前后的更多上下文。

### 复现（Reproduce）

复现是用明确步骤再次得到同样的问题。

一个问题越容易复现，就越容易定位和验证。

### 排错（Troubleshooting）

排错是根据现象和线索，逐步定位原因，并用明确操作验证判断的过程。

排错不是盲目重试，而是有步骤地缩小问题范围。

## 3. 今日动手任务

### 任务 1：路径不存在

命令：

```powershell
Get-ChildItem .\not-exist-folder
```

这类错误说明：命令存在，但传给命令的目标路径找不到。

辅助验证：

```powershell
Get-Location
Test-Path .\not-exist-folder
Resolve-Path .
```

### 任务 2：命令不存在

命令：

```powershell
not-a-real-command
```

这类错误说明：Shell 连要执行的命令都找不到。

辅助验证：

```powershell
Get-Command not-a-real-command
```

### 任务 3：Java 类不存在

命令：

```powershell
java NotExistClass
$LASTEXITCODE
```

这类错误说明：`java` 命令可能已经被找到并执行，但 Java 运行时找不到指定的类。

辅助验证：

```powershell
Get-Command java
java -version
$LASTEXITCODE
```

## 4. 三类错误对比

| 错误类型 | 失败点 | 关键线索 | 验证方式 |
| --- | --- | --- | --- |
| 路径不存在 | 命令存在，目标路径找不到 | 路径名、当前目录 | `Get-Location`、`Test-Path` |
| 命令不存在 | Shell 找不到命令 | 命令名、not recognized | `Get-Command` |
| Java 类不存在 | Java 运行时找不到类 | 类名、ClassNotFound 相关信息 | `Get-Command java`、`java -version` |

## 5. 固定排错流程

```text
1. 记录我执行的完整命令。
2. 保留完整错误信息。
3. 找出最重要的关键词：路径、命令名、类名、异常名、端口号。
4. 确认当前路径和环境。
5. 提出最可能的原因。
6. 用一条明确命令验证这个原因。
7. 如果不成立，再换下一个假设。
```
